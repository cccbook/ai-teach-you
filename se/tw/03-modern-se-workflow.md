# 3. 現代軟體工程的完整流程

## 從想法到上線

現代軟體工程是一個複雜的協作過程。讓我們鳥瞰整個流程，理解每個環節如何與 AI 協作。

## 第一階段：需求與分析

### 傳統做法
產品經理寫 PRD，工程師問問題確認，來回可能耗費數週。

### AI 輔助做法
```
你 → AI：「我有一個關於 [功能描述] 的想法，請問需要考慮哪些問題？」
AI → 你：提供檢查清單、技術考量、潛在風險

你 → AI：「請將以下需求轉化為技術術語：[PRD內容]」
AI → 你：轉化後的需求，可能發現模糊之處
```

### AI 在這個階段能幫你
- 識別需求中的歧義
- 列出技術上需要考慮的事項
- 生成需求檢查清單
- 估計功能複雜度

## 第二階段：系統設計

### 架構設計
AI 擅長：
- 根據需求推薦架構模式
- 畫出系統架構圖（mermaid 格式）
- 解釋不同架構的優缺點
- 生成 API 設計草稿

### 資料建模
```
你 → AI：「我需要設計一個電子商務資料庫，請問需要哪些實體？」
AI → 你：users, products, orders, order_items, categories...
```

## 第三階段：開發編碼

這是 AI 最有價值的階段：

### 日常輔助
| 任務 | 沒有 AI | 有 AI |
|------|---------|-------|
| 查語法 | 查文檔 3-5 分鐘 | 即時回答 |
| 寫樣板 | 複製貼上 + 修改 | 直接生成 |
| 寫測試 | 手動撰寫 | 自動生成基礎測試 |
| 除錯 | 打斷點 + 追蹤 | 描述問題 → AI 分析 |
| 理解代碼 | 逐行閱讀 | AI 解釋 + 提問 |

### 實用工作流
```
1. 任務拆解
   你：「我要實現用戶登入功能」
   AI：「可以拆分為：前端表單、后端 API、資料庫查詢、Session 管理...」

2. 逐一實現
   你：「現在請幫我生成 JWT 驗證的中間件」
   AI：[生成代碼]

3. 整合調試
   你：「這些片段整合後出現 [錯誤]，可能的原因？」
   AI：[分析]
```

## 第四階段：測試

### AI 在測試中的角色
- **生成測試用例**：根據功能描述生成測試覆蓋
- **發現邊界條件**：幫你思考「如果用戶輸入 X 會怎樣」
- **解釋測試失敗**：描述錯誤含義和修復方向
- **性能測試建議**：推薦壓測工具和策略

### AI 輔助測試示例
```
你：「請為這個函數生成邊界測試用例」
AI → 你：
- 空輸入
- 非常大的輸入
- 特殊字元
- SQL 注入嘗試
- XSS 攻擊嘗試
```

## 第五階段：部署與維運

### 部署自動化
AI 能幫你：
- 撰寫 Dockerfile
- 配置 CI/CD 流程
- 撰寫部署腳本
- 撰寫回滾腳本

### 監控與告警
```
你：「如何監控這個微服務？需要哪些指標？」
AI → 你：
- 回應時間 (latency)
- 錯誤率 (error rate)
- 請求量 (throughput)
- 資源使用率 (CPU, memory)
- 自定義業務指標
```

## 關鍵認知

AI 不是魔法師，而是一個：
- **極快的實習生**：能幫你做重複性工作
- **耐心的老師**：隨時回答你的問題
- **多語言翻譯**：將你的想法轉化為代碼

你的價值在於：
- 理解要解決的問題
- 判斷 AI 的建議是否正確
- 整合所有零件成完整系統
- 為最終結果負責
