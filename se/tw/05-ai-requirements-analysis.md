# 5. 用 AI 釐清需求

## 需求是軟體開發的根基

「需求不清，後患無窮」。一個模糊的需求，即使 AI 能生成代碼，做出來的東西也可能完全不是你想要的。

## 為什麼 AI 能幫助需求分析

AI 擅長：
- 識別描述中的歧義
- 列舉可能被遺漏的場景
- 將模糊想法結構化
- 提供不同角度的思考

## 需求分析對話模式

### 模式一：想法澄清

```
你：
我想做一個筆記應用，用戶可以寫筆記、分類筆記、搜尋筆記。

AI：
太好了！在你開始之前，我想幫你確認幾個問題：

功能範圍：
1. 用戶體系 - 需要登入嗎？還是本地使用？
2. 筆記格式 - 純文字？支援 Markdown？圖片？
3. 分類方式 - 資料夾？標籤？兩者都要？
4. 搜尋功能 - 關鍵字搜尋？全文搜尋？兩者？
5. 同步機制 - 單設備？多設備同步？

非功能需求：
6. 離線支援 - 需要離線使用嗎？
7. 資料安全 - 需要加密嗎？
8. 擴展性 - 以後可能加協作功能嗎？

這些問題的答案會影響架構設計和技術選型。
```

### 模式二：場景列舉

```
你：
幫我思考一個用戶登入系統需要考慮哪些場景。

AI：
基本場景：
✓ 正確的帳號密碼登入成功
✓ 錯誤密碼登入失敗
✓ 不存在的帳號

進階場景：
○ 密碼多次錯誤後鎖定
○ 記住登入狀態 (Remember Me)
○ 第三方登入 (Google, GitHub)
○ 社交帳號綁定

安全場景：
○ 暴力破解防護
○ Session 過期處理
○ 異地登入通知
○ 裝置管理

邊界場景：
○ 密碼大小寫敏感性
○ 特殊字元的處理
○ 空密碼的處理
○ 超長密碼的處理
```

### 模式三：需求轉化

```
你：
將以下需求轉化為技術規格：
「用戶可以上傳照片，在照片上加上標籤，其他人可以透過標籤搜尋照片」

AI：
## 技術規格

### 核心功能
| 功能 | 描述 | 優先級 |
|------|------|--------|
| 照片上傳 | 支援格式、大小限制、壓縮 | P0 |
| 標籤系統 | 多標籤支援、標籤自動建議 | P0 |
| 搜尋功能 | 標籤檢索、組合搜尋 | P0 |

### API 設計
```
POST /api/photos          - 上傳照片
GET  /api/photos?tags=xx  - 依標籤搜尋
PUT  /api/photos/:id/tags - 更新標籤
```

### 資料模型
```
Photo {
  id: UUID
  url: string
  user_id: UUID
  created_at: timestamp
}

Tag {
  id: UUID
  name: string
}

PhotoTag {
  photo_id: UUID
  tag_id: UUID
}
```

### 非功能需求
- 照片大小限制：10MB
- 支援格式：JPEG, PNG, WebP
- 縮圖生成：是
- CDN 加速：是
```

## 有效的提問技巧

### 技巧一：提供上下文

```
❌ 不好：「帮我设计数据库」

✅ 好：「我正在开发一个博客系统，使用 PostgreSQL，
     预计有 1000 个作者，每个作者平均 50 篇文章，
     请帮我设计数据库 Schema」
```

### 技巧二：說明約束條件

```
❌ 不好：「这个功能怎么实现？」

✅ 好：「在 512MB 内存的服务器上，如何高效处理
     10GB 的日志文件？请考虑内存限制」
```

### 技巧三：明確輸出格式

```
❌ 不好：「告诉我需要考虑什么」

✅ 好：「请用 Markdown 表格列出电商购物车的
     所有功能点，包含：功能名称、描述、验收标准」
```

### 技巧四：先給框架再補充

```
你：帮我列出用户管理系统的功能模块
AI：[列出模块]

你：针对「权限管理」模块，详细说明每个权限级别的定义
AI：[详细说明]
```

## 需求文件模板

讓 AI 幫你生成需求文件：

```
請幫我生成一個功能需求文件模板，包含：
1. 功能概述
2. 用戶故事
3. 功能詳情
4. 驗收標準
5. 非功能需求
6. 依賴關係
7. 風險評估
```

## 避免的陷阱

### 陷阱一：需求範圍蔓延

```
你：加一個簡單的功能
AI：好的 [生成代碼]
你：咦，既然都能這樣了，不如再加個 XX 功能...
AI：可以 [修改代碼]
你：再優化一下 YY...


問題：沒有明確的範圍邊界
解決：每次確認前問「這是 scope 內還是 scope 外？」
```

### 陷阱二：假設 AI 知道你的想法

```
你心裡想的：「這個當然要有」
AI 理解的：「你說的就是字面意思」

解決：明確說出你的假設，讓 AI 幫你驗證
```

## 實踐練習

設計一個 AI 輔助的需求分析對話：

```
主題：在线预约理发系统

目标：
1. 用 AI 幫你列举需要考虑的场景
2. 生成完整的功能清单
3. 转换为技术规格
```

在開始下一章之前，先試著用這個流程分析你當前的一個專案需求。
