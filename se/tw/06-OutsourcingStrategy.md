# 第 6 章：外包策略

> Agent 做不到的，用外包

---

## 6.1 外包決策框架

> 檔案：[06-1-OutsourcingDecision.md](../_code/06/06-1-OutsourcingDecision.md)

### 什麼該外包？

| 類別 | 外包項目 | 原因 |
|------|----------|------|
| 設計 | UI/UX 設計 | AI 審美有限 |
| 架構 | 雲端架構顧問 | 需要專業經驗 |
| 安全 | 滲透測試 | 需要專業認證 |
| 法務 | 合規審查 | 需要專業資格 |

### 什麼該自己做？

| 類別 | 原因 |
|------|------|
| 核心功能 | 智慧財產權 |
| 商業邏輯 | 競爭優勢 |
| 客戶資料 | 隱私安全 |
| AI Prompt | 累積經驗 |

---

## 6.2 UI/UX 設計外包

> 檔案：[06-2-UIDesignOutsource.md](../_code/06/06-2-UIDesignOutsource.md)

### 外包平台

| 平台 | 費用 | 特點 |
|------|------|------|
| Fiverr | $50-500 | 快速、便宜 |
| 99designs | $300-1500 | 專業競稿 |
| Toptal | $5000+ | 頂級設計師 |
| Dribbble | $200-2000 | 高品質設計師 |

### 外包流程

```
1. 準備設計 brief
   - 產品定位
   - 目標用戶
   - 功能清單
   - 參考網站

2. 選擇設計師
   - 查看作品集
   - 確認風格契合
   - 討論時程與費用

3. 交付與驗收
   - 取得設計稿
   - 取得 source file
   - 確認智慧財產權
```

## 6.3 雲端架構顧問

### 顧問選擇要點

- 相關產業經驗
- 雲端認證（AWS/Azure/GCP）
- 收費方式（時薪/專案）

### 諮詢時機

| 階段 | 諮詢內容 |
|------|----------|
| 專案開始 | 架構建議 |
| 遇到瓶颈 | 效能優化 |
| 擴大規模 | 擴展策略 |
| 安全審計 | 合規建議 |

---

## 6.4 資安與滲透測試

> 檔案：[06-3-SecurityOutsource.md](../_code/06/06-3-SecurityOutsource.md)

### 滲透測試時機

- 上線前
- 重大功能上線
- 每半年定期
- 發生資安事件後

### 資安外包項目

| 項目 | 頻率 | 費用 |
|------|------|------|
| 滲透測試 | 半年/次 | 5-20萬 |
| 原始碼審計 | 上線前 | 3-10萬 |
| 弱點掃描 | 每月 | 1-3萬 |
| 合規顧問 | 必要時 | 5-30萬 |

---

## 6.5 法務與合規

### 需要律師的情況

- 隱私權政策
- 使用者條款
- 資料處理協議
- 產業合規（金融、醫療）

---

## 6.6 本章總結

外包策略關鍵原則：

1. **AI 擅長的自己做**：程式碼、測試、部署
2. **專業的外包**：設計、資安、法務
3. **核心的自己掌握**：商業邏輯、客戶資料
4. **適度外包**：不要過度依賴外包
