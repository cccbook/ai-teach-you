# 第 1 章：案例：小型敏捷專案

> 一人團隊電商從零到上線

本章以一個真實案例，展示如何使用 AI Agent 打造一個小型電商網站。從構想到上線，僅用 8 週。

---

## 1.1 案例背景

### 產品構想

創辦人小李是一位獨立開發者，他想要建立一個簡單的禮品電商網站「GiftNow」：

- **產品**：精選禮品推薦平台
- **目標客群**：25-40 歲上班族，選擇禮物困擾
- **核心功能**：禮品推薦、購物車、訂單管理、後台庫存
- **上線時程**：8 週

### 為什麼選擇一人團隊？

小李評估後發現：

| 選項 | 成本 | 時間 | 風險 |
|------|------|------|------|
| 傳統外包 | 30 萬起 | 3-6 個月 | 溝通成本高 |
| 兼職團隊 3 人 | 每月 10 萬 | 4 個月 | 管理負擔 |
| 一人 + AI Agent | 每月 2 萬 | 2 個月 | 技術極限 |

結論：選擇一人團隊，最大化 AI 能力覆蓋範圍。

## 1.2 團隊運作方式

檔案：[01-1-OneManTeam.md](../_code/01/01-1-OneManTeam.md)

### 角色分配

傳統團隊需要 PM、Designer、Frontend、Backend、DevOps、QA 等角色。在一人團隊中：

| 傳統角色 | 一人團隊做法 |
|----------|--------------|
| PM | 自己擔任，用 AI 輔助需求分析 |
| Designer | 外包給 Fiverr 設計師 |
| Frontend | AI (Cursor) 生成 React 程式碼 |
| Backend | AI 生成 FastAPI 程式碼 |
| DevOps | Vercel 一鍵部署 |
| QA | AI 生成測試 + 自動化 |
| 監控 | AI 監控服務 |

> 檔案：[01-1-OneManTeam.md](../_code/01/01-1-OneManTeam.md)

### 每日工作流程

```
早上 9:00  - 查看監控儀表板 + 處理异常
早上 9:30  - 與 AI Agent 對話，推進當日開發任務
中午 12:00 - 午餐 + 思考產品方向
下午 2:00  - 審核 AI 生成的程式碼
下午 3:00  - 外包溝通（設計師/顧問）
下午 5:00  - 部署測試環境
晚上 8:00  - 回饋收集與產品迭代
```

## 1.3 技術選型

> 檔案：[01-2-TechStack.md](../_code/01/01-2-TechStack.md)

### 前端技術

| 選項 | 優點 | 缺點 | 選擇理由 |
|------|------|------|----------|
| Next.js | SSR SEO 好、生態大 | 學習曲線 | AI 擅長、社群資源多 |
| Vue | 簡單易學 | 生態較小 | - |
| 純 HTML | 最簡單 | 難維護 | - |

**選擇：Next.js 14 + Tailwind CSS**

### 後端技術

| 選項 | 優點 | 缺點 | 選擇理由 |
|------|------|------|----------|
| Next.js API | 同一專案 | 擴展性有限 | 最快速 |
| FastAPI | 效能高、類型安全 | 需分開部署 | AI 擅長 Python |
| Node.js | 生態大 | TypeScript 複雜 | - |

**選擇：FastAPI + SQLite（初期）→ PostgreSQL（後期）**

### 部署平台

| 平台 | 費用 | 特點 | 適合 |
|------|------|------|------|
| Vercel | 免費額度 | 零設定部署 | Next.js |
| Railway | 免費額度 | 彈性 | 後端服務 |
| Render | 免費額度 | 簡單 | 快速啟動 |

**選擇：Vercel (前端) + Railway (後端)**

### 完整技術棧

```
┌─────────────────────────────────────────┐
│             GiftNow 技術棧              │
├─────────────────────────────────────────┤
│  前端: Next.js 14 + Tailwind CSS       │
│  後端: FastAPI + SQLAlchemy            │
│  資料庫: SQLite → PostgreSQL           │
│  部署: Vercel + Railway                │
│  監控: Sentry + LogSnag                │
│  AI: Claude Code + Cursor              │
└─────────────────────────────────────────┘
```

## 1.4 開發流程日誌

> 檔案：[01-3-DevLog.md](../_code/01/01-3-DevLog.md)

### 第 1-2 週：規劃與設計

| 日期 | 任務 | AI 輔助 | 產出 |
|------|------|---------|------|
| Day 1-2 | 市場研究 | Claude 分析競品 | 競品分析報告 |
| Day 3-4 | 需求定義 | AI 生成 User Story | 需求文件 |
| Day 5-6 | UI/UX 設計 | 外包 Fiverr | Figma 設計稿 |
| Day 7-10 | 系統規劃 | AI 產出架構圖 | 系統架構文檔 |

**AI Prompt 範例**：

```
請幫我分析三家禮品電商網站的特色：
1. GiftNow.io
2. PresentBox
3. Fancy

請用表格呈現功能比較、重點差異化、建議。
```

### 第 3-4 週：MVP 開發

| 日期 | 功能 | AI 程式碼產出 | 人工審核 |
|------|------|---------------|----------|
| Day 11-12 | 首頁框架 | 90% | 10% 調整 |
| Day 13-14 | 商品列表 | 85% | 15% 優化 |
| Day 15-16 | 購物車 | 80% | 20% 修正 |
| Day 17-18 | 使用者登入 | 75% | 25% 安全檢查 |
| Day 19-20 | API 串接 | 90% | 10% 測試 |
| Day 21-28 | 整合測試 | 70% | 30% Debug |

**核心開發 Prompt**：

```
請用 FastAPI 建立一個商品 API，包含：
1. GET /products - 商品列表
2. GET /products/{id} - 商品詳情
3. POST /orders - 創建訂單

使用 SQLAlchemy + SQLite。
```

### 第 5-6 週：優化與測試

| 任務 | AI 輔助 | 說明 |
|------|---------|------|
| 單元測試 | 90% | AI 生成 pytest 測試 |
| 整合測試 | 70% | 人工設計場景 |
|效能優化|60%|AI 建議 + 人工判斷|
|安全檢查|50%|AI 掃描 + 外包審計|

### 第 7-8 週：部署上線

| 日期 | 任務 | 產出 |
|------|------|------|
| Day 43-45 | 生產環境部署 | Vercel + Railway 上線 |
| Day 46-50 | 使用者測試 | 10 位測試用戶 |
| Day 51-54 | 問題修復 | Bug 修復清單 |
| Day 55-56 | 正式上線 | GiftNow 網站上線 |

## 1.5 部署設定

> 檔案：[01-4-DeployConfig.md](../_code/01/01-4-DeployConfig.md)

### 前端部署（Vercel）

```yaml
# vercel.json
{
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "framework": "nextjs"
}
```

```yaml
# railway.json
{
  "build": {
    "builder": "NIXPACKS"

  "run": {
    "command": "uvicorn main:app --host 0.0.0.0 --port $PORT"

}
```

## 1.7 成果與學習

### 上線成果

| 指標 | 數值 |
|------|------|
| 開發時間 | 8 週 |
| 總成本 | 16 萬（含外包 + 雲端）|
| 功能數量 | 12 個核心功能 |
| Bug 數量 | 上線時 3 個 minor |
| 使用者滿意度 | 4.2/5 |

### 關鍵學習

1. **AI 是強大的開發夥伴**：80% 的程式碼由 AI 生成
2. **外包彌補 AI 不足**：UI/UX 設計交給專業設計師
3. **快速迭代勝於完美**：MVP 先上線，再持續優化
4. **監控不可省略**：及早發現問題，避免用戶流失

### 失敗與改進

| 失敗點 | 改進措施 |
|--------|----------|
| 最初選錯資料庫 | 從 SQLite 遷移 PostgreSQL |
| 測試不夠完整 | 加入自動化 E2E 測試 |
| 沒有備份機制 | 每日自動備份 |

## 1.8 本章總結

本章展示了如何使用「一人團隊 + AI Agent」模式，在 8 週內從零打造一個電商網站。關鍵要點：

1. **AI 處理 80% 開發工作**：程式碼生成、測試、部署腳本
2. **外包處理 20% 非技術工作**：UI/設計、資安顧問
3. **人類專注決策**：產品方向、架構選擇、品質把關
