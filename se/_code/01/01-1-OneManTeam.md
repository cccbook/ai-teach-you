## 角色分配對照表

| 傳統角色 | 一人團隊做法 | AI/工具輔助度 |
|----------|--------------|---------------|
| PM | 自己擔任 | AI 輔助需求分析 |
| UI Designer | 外包 Fiverr | - |
| UX Researcher | 自己 + AI 問卷 | AI 輔助分析 |
| Frontend Dev | AI (Cursor) | 90% AI 生成 |
| Backend Dev | AI (Claude Code) | 90% AI 生成 |
| DevOps | AI + Vercel | 80% 自動化 |
| QA | AI 生成測試 | 70% AI 生成 |
| 監控 | AI 監控服務 | 60% AI |

---

## 每日工作時間表

```
09:00 - 監控檢查（15 分鐘）
      查看 Sentry 錯誤、Vercel 儀表板
      處理昨夜異常

09:30 - AI 開發時間（3 小時）
      與 Claude/Cursor 對話
      生成程式碼、審核、修正
      推進當日開發任務

12:00 - 午餐休息

14:00 - 程式碼審核（1 小時）
      審查 AI 產出
      人工把關品質

15:00 - 外包溝通（1 小時）
      與設計師對接
      技術顧問諮詢

16:00 - 測試與部署（1 小時）
      部署到測試環境
      執行測試案例

17:00 - 文件與回饋（1 小時）
      更新 README
      回覆使用者回饋
      規劃明日任務
```

## AI 協作 Prompt 範本

### 晨間任務分配

```
今天我要完成「GiftNow」電商網站的購物車功能。

請帮我：
1. 先回顧現有程式碼結構
2. 列出需要修改的檔案
3. 從最重要的部分開始開發
```

### 程式碼審核

```
請檢查以下程式碼的安全性問題：

[貼上程式碼]

重點檢查：
- SQL Injection
- XSS 攻擊
- 權限控制
```

### Bug 修復

```
遇到這個錯誤：
Error: Cannot read property 'map' of undefined

請帮我：
1. 分析可能原因
2. 找出問題所在
3. 提供修復方案
```

## 品質把關清單

每次 AI 生成程式碼後，人工審核項目：

- [ ] 程式碼邏輯正確
- [ ] 命名清晰易懂
- [ ] 安全性檢查
- [ ] 錯誤處理
- [ ] 效能考量
- [ ] 測試覆蓋

## 關鍵成功因素

1. **清晰的產品願景**：知道要做什麼
2. **有效的 AI Prompt**：知道怎麼問 AI
3. **嚴格的品質把關**：知道審核什麼
4. **適時的外包支援**：知道什麼該外包
