# 團隊選擇決策表

> 來源：第 2 章「為何一人團隊」

---

## 何時選擇什麼團隊規模？

### 評估維度

| 維度 | 一人團隊 | 小團隊(2-5人) | 大團隊(10+人) |
|------|----------|---------------|---------------|
| 預算 | < 5萬/月 | 10-50萬/月 | 50萬+/月 |
| 時程 | < 3個月 | 3-12個月 | 6-24個月 |
| 複雜度 | 簡單 | 中等 | 複雜 |
| 用戶規模 | < 1000 | 1000-10萬 | 10萬+ |
| 法規需求 | 低 | 中等 | 高 |

### 決策流程圖

```
開始
  │
  ▼
專案複雜度？
  │ 低 → 一人團隊
  │
  ▼
法規需求？
  │ 高 → 大團隊
  │
  ▼
時程緊迫？
  │ 是 → 一人團隊
  │
  ▼
預算有限？
  │ 是 → 一人團隊
  │
  ▼
選擇小團隊
```

### 成本比較

| 團隊類型 | 月成本 | 年成本 | 適合階段 |
|----------|--------|--------|----------|
| 一人+AI | 2-5萬 | 24-60萬 | MVP、驗證 |
| 小團隊 | 10-30萬 | 120-360萬 | 成長 |
| 大團隊 | 50-100萬 | 600-1200萬 | 規模化 |
