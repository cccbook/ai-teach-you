# 程式類書籍寫作 Skill

> **注意**：本文件使用繁體中文撰寫。

## 概述

本 Skill 提供撰寫程式設計類書籍的完整工作流程，涵蓋程式碼範例提取、測試、索引建立等環節。適用於任何程式設計相關書籍的寫作專案。

## 核心原則

1. **理論與實踐結合**：每個概念都應配有可執行的程式碼範例
2. **程式碼可測試**：所有提取的程式碼必須經過測試驗證
3. **交叉引用**：書籍章節與程式碼檔案相互連結
4. **術語統一**：建立統一的專有名詞索引

## 工作流程

### 第一階段：準備階段

#### 1. 了解書籍結構

```
book_dir/
├── tw/                    # 繁體中文版
│   ├── 01-xxx.md         # 第1章
│   ├── 02-xxx.md         # 第2章
│   └── ...
├── en/                    # 英文版（如有）
└── _code/                # 程式碼目錄
    ├── 01/               # 第1章程式碼
    ├── 02/               # 第2章程式碼
    └── ...
```

#### 2. 建立程式碼目錄結構

根據書籍章節數建立對應的程式碼目錄：

```bash
mkdir -p book_dir/_code/{01,02,03,...,14}
```

#### 3. 建立程式碼清單 (code_list.md)

在開始撰寫書籍之前，先建立 `book_dir/_code/code_list.md`，規劃每章的程式碼範例：

```markdown
# 程式碼範例清單

## 第 1 章：XXX
- `01-1-example.py` - 範例說明
- `01-2-example.py` - 範例說明

## 第 2 章：YYY
- `02-1-example.c` - 範例說明
```

**建立清單的好處**：
- 先規劃所有程式碼，確保每章都有實作範例
- 確認程式碼涵蓋足夠的主題
- 方便後續追蹤進度
- 可作為書籍的附錄參考

### 第二階段：程式碼提取

#### 1. 從 Markdown 提取程式碼塊

對於每章的每個程式碼塊：

1. 識別程式碼塊語言（c, python, assembly, java 等）
2. 建立帶編號的程式碼檔案：`XX-Y-description.ext`
3. 複製程式碼內容（去除 triple backticks）
4. 處理語言特殊語法：
   - C/C++: 使用 `//` 或 `/* */` 註解
   - Python: 使用 `#` 註解
   - Assembly: 使用 `;` 註解

#### 2. 程式碼檔案命名規則

```
格式：{章節號}-{順序}-{描述}.{副檔名}
範例：
  01-1-hello.c        # 第1章第1個C程式
  02-3-parser.py      # 第2章第3個Python程式
  05-5-x86_64.s       # 第5章第5個組合語言程式
```

#### 3. 注意事項

- 跳過不完整的程式碼片段
- 跳過需要特殊執行環境但本地沒有的語言（如 Prolog, Haskell 互動式環境）
- 跳過純示範性的命令輸出塊

### 第三階段：程式碼測試

#### 1. 可測試語言

以下語言應進行測試：

- **C**: 使用 `gcc` 編譯執行
- **Python**: 使用 `python3` 直接執行
- **Shell/Bash**: 使用 `bash` 執行
- **Verilog**: 使用 `icarus + vvp` 執行

#### 2. 測試命令

```bash
# C 程式
gcc -o output source.c && ./output

# Python 程式
python3 script.py

# Shell 腳本
bash script.sh
```

#### 3. 測試策略

- 每個可執行檔都應執行驗證
- 檢查輸出是否符合預期
- 修正語法錯誤直至通過

### 第四階段：連結添加

#### 1. 連結格式

在每個程式碼塊前新增 Markdown 連結：

```markdown
[程式檔案：01-1-hello.c](../_code/01/01-1-hello.c)
```c
#include <stdio.h>
int main() {
    printf("Hello!\n");
    return 0;
}
```
```

#### 2. 相對路徑規則

- 書籍檔案在 `tw/` 目錄
- 程式碼檔案在 `../_code/XX/` 目錄
- 使用相對路徑：`../_code/01/01-1-hello.c`

#### 3. 連結放置位置

- **格式**：`[程式檔案：{檔案名稱}](../_code/{章節}/{檔案名稱})`
- **位置**：程式碼塊 **前面**，緊鄰程式碼

### 第五階段：專有名詞索引

#### 1. 建立索引目錄

```
book_dir/tw/index/
├── README.md           # 索引主頁面
├── lisp.md            # LISP 詳細說明
├── compiler.md        # 編譯器詳細說明
└── ...
```

#### 2. 索引主頁面格式

```markdown
# 專有名詞索引

## 程式語言

* [LISP](lisp.md) : 1958 年由 John McCarthy 發明的函數式語言。
* [C 語言](c.md) : 1972 年由 Dennis Ritchie 發明的系統程式語言。

## 編譯器與直譯器

* [編譯器 (Compiler)](compiler.md) : 將原始碼翻譯為機器碼的工具。
```

#### 3. 詳細說明檔案結構

每個專有名詞檔案應包含：

```markdown
# {專有名詞}

## 概述
（簡要說明）

## 歷史
- 發明時間
- 發明者
- 重要里程碑

## 核心概念
（說明 + 程式碼範例）

## 為什麼重要
（應用場景、優勢）

## 參考資源
```

## 常見問題處理

### Q1: 程式碼檔案 LSP 報錯

**原因**：某些程式碼是示範片段，不是完整程式

**解決**：
- 檢查程式碼是否完整可編譯
- 片段程式碼不需修復，標記為「示範程式碼」

### Q2: 連結路徑錯誤

**問題**：`sp/_code/01/01-1-hello.c` 無法存取

**解決**：使用相對路徑 `../_code/01/01-1-hello.c`

### Q3: 某些語言無法測試

**原因**：未安裝對應執行環境

**解決**：
- 跳過測試，繼續處理其他檔案
- 在檔案中新增註解說明執行要求

2. **連結新增**：批次新增 Markdown 連結
3. **術語檔案**：並行建立多個術語檔案

1. ✅ 所有程式碼塊已提取為獨立檔案
2. ✅ 所有可測試程式碼已驗證通過
3. ✅ 所有程式碼塊前都有對應連結
4. ✅ 專有名詞索引已建立
5. ✅ 程式碼目錄結構與章節對應

## 擴展使用

本 Skill 可應用於：

- 作業系統書籍
- 編譯原理書籍
- 網路程式設計書籍
- 人工智慧書籍
- 任何包含程式碼範例的技術書籍

只需根據具體書籍調整：
- 章節數量
- 程式碼目錄深度
- 專有名詞列表
