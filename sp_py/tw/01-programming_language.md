# 1. 高階語言

## 1.1 高階語言的歷史與發展

### 1.1.1 機器語言的起源

電子計算機於 1940 年代誕生，早期程式設計必須直接使用機器語言（Machine Language）。機器語言是電腦處理器能夠直接執行的二進制指令編碼，每個指令對應處理器中的一個微操作。

**機器語言的特性：**

| 特性 | 說明 |
|------|------|
| 二進制表示 | 指令和資料都以 0/1 位元序列表示 |
| 處理器相依 | 不同 CPU 架構（如 x86、ARM）有完全不同的指令集 |
| 難以閱讀 | 人類無法直觀理解二進制編碼的意義 |
| 難以維護 | 修改程式極易出錯，錯誤難以偵測 |

以 x86 架構為例，以下是兩個簡單的機器語言指令：

```
B8 01 00    ; 將數值 1 載入 AX 暫存器（操作碼 B8 + 立即值 0001）
89 C3        ; 將 AX 的值複製到 BX 暫存器（操作碼 89 + 目的地 BX）
```

這些十六進位數字對應處理器內部的電路訊號。`B8` 是「將立即值載入 AX」的 opcode，而 `01 00` 表示數值 1（低位元組在前的小端序）。

### 1.1.2 組合語言的革命

1950 年代，組合語言（Assembly Language）的發明標誌著程式設計的重大進步。組合語言用助記憶符號（Mnemonic）代替冰冷的二進位碼，大幅提升了程式的可讀性。

**組合語言的設計原理：**

```
機器語言：  B8 01 00
組合語言：  mov ax, 1     ; 將 1 載入 AX 暫存器
```

每個組合語言語法直接對應一條機器指令，翻譯過程稱為「組譯」（Assembly）。組譯器（Assembler）將助記憶符號轉換為機器碼。

組合語言的關鍵特性：

1. **一對一對應**：每條組合語言語句對應一條機器指令
2. **使用標籤**：用符號名稱代替絕對記憶體位址
3. **巨集支援**：可定義可重複使用的程式碼片段
4. **跨平台困難**：不同 CPU 架構有不同的組合語言語法

### 1.1.3 高階語言的誕生

1950 年代末期，高階語言的出現徹底改變了程式設計的面貌。高階語言提供抽象層次，使程式設計師能以更接近人類自然語言的方式表達演算法。

**第一個高階語言：Fortran**

1957 年，IBM 的 John Backus 團隊發明了 Fortran（Formula Translation），這是第一個成功的高階語言。Fortran 專為科學和工程計算設計，其核心設計理念：

- 讓科學家能直接表達數學公式
- 編譯器自動產生高效機器碼
- 接近數學表示法的語法

```fortran
C 計算圓面積
      PROGRAM AREA
      REAL R, A
      READ(*,*) R
      A = 3.14159 * R * R
      PRINT *, 'AREA =', A
      END
```

Fortran 的創新之處在於它首次實現了「軟體危機」的解決方案——將程式設計從機器導向轉變為問題導向。

### 1.1.4 關鍵高階語言的歷史貢獻

**ALGOL (1958-1968)**

ALGOL（ALGOrithmic Language）奠定了結構化程式設計的基礎，引入了區塊結構（Block Structure）、遞迴函數、和 BNF 文法描述。這些概念影響了後續幾乎所有程式語言的設計。

**C 語言 (1972)**

Dennis Ritchie 在貝爾實驗室發明了 C 語言，其設計哲学：

1. **低階存取能力**：可直接操作記憶體和暫存器
2. **高效能**：產生的機器碼接近組合語言效率
3. **可攜性**：透過標準函式庫隔離硬體差異
4. **簡潔語法**：保持最小的語言核心

C 語言的誕生與 Unix 作業系統的開發密切相關。Ritchie 和 Thompson 選擇用 C 重寫 Unix，證明了高階語言同樣能夠開發作業系統。

**C++ (1983)**

Bjarne Stroustrup 設計的 C++ 在 C 語言基礎上加入物件導向特性。其設計目標是「C with Classes」——在保持 C 語言效率的同時，引入封裝、繼承、多型等 OOP 概念。

**Java (1995)**

Sun Microsystems 發明的 Java 帶來了革命性的「Write Once, Run Anywhere」理念：

- Java 程式編譯為位元組碼（Bytecode）
- 由 Java 虛擬機（JVM）在不同平台上解釋執行
- 自動記憶體管理（垃圾回收）
- 豐富的標準函式庫

**Python (1991)**

Guido van Rossum 設計的 Python 以簡潔易學著稱，其核心哲學：

- 程式碼可讀性優先
- 一個問題只有一種最佳解決方式
- 內建電池（Batteries Included）哲學

**Go (2009)**

Google 發明的 Go 語言專為現代伺服器程式設計：

- 輕量並行（goroutine + channel）
- 快速編譯
- 記憶體安全
- 靜態型別但動態 feel

**Rust (2010)**

Mozilla 設計的 Rust 追求記憶體安全而無需垃圾回收：

- 所有權系統（Ownership）防止記憶體錯誤
- 借用檢查器（Borrow Checker）確保執行緒安全
- 零成本抽象

### 1.1.5 語言設計的理論基礎

程式語言設計涉及多個理論領域：

**語言的抽象層次**

```
高層次抽象（人類友好）
    │
    ▼
╔═══════════════════════════════════════════════════════╗
║  應用領域語言 (DSL)                                  ║
║  Python, SQL, MATLAB                                  ║
╠═══════════════════════════════════════════════════════╣
║  通用高階語言                                         ║
║  Java, C++, Go, Rust                                  ║
╠═══════════════════════════════════════════════════════╣
║  系統程式語言                                         ║
║  C, C++, Rust                                         ║
╠═══════════════════════════════════════════════════════╣
║  組合語言                                             ║
║  x86 ASM, ARM ASM, RISC-V ASM                         ║
╠═══════════════════════════════════════════════════════╣
║  機器語言                                             ║
║  處理器指令集                                          ║
╚═══════════════════════════════════════════════════════╝
    │
    ▼
低層次抽象（機器執行）
```

**語言的 평가模型（Evaluation Model）**

每種語言都有其獨特的計算模型，決定了程式的執行語意：

| 模型 | 代表語言 | 特點 |
|------|----------|------|
| 命令式 | C, Pascal, Fortran | 逐步改變狀態 |
| 函數式 | Haskell, LISP, ML | 數學函數求值，無副作用 |
| 邏輯式 | Prolog | 自動定理證明 |
| 物件導向 | Smalltalk, Java | 訊息傳遞與封裝 |

## 1.2 程式語言的分類

### 1.2.1 命令式語言（Imperative Language）

命令式語言是最傳統也是最廣泛使用的程式設計範式。其核心思想是：**程式是由一連串改變程式狀態的命令組成**。

**命令式語言的組成要素：**

1. **變數（Variables）**：儲存可變動的資料值
2. **賦值語句（Assignment）**：改變變數的值
3. **控制結構（Control Structures）**：順序、選擇、迴圈
4. **程序（Procedures）**：程式碼封裝與重用

**狀態機模型**

命令式語言的語意可用狀態轉換表示：

```
初始狀態 ──命令1──→ 狀態1 ──命令2──→ 狀態2 ──...
                      │                    │
                      ▼                    ▼
                    輸出                  輸出
```

```c
// C 語言 - 命令式範例
int sum = 0;           // 狀態：sum = 0
for (int i = 1; i <= 10; i++) {
    sum += i;          // 狀態改變：sum = sum + i
}
printf("%d\n", sum);  // 輸出最終狀態
```

**命令式語言的優缺點：**

| 優點 | 缺點 |
|------|------|
| 符合硬體執行模型 | 難以推理程式行為 |
| 直覺、容易理解 | 共享狀態導致並行困難 |
| 高效率 | 難以驗證正確性 |
| 控制力強 | 容易產生副作用 |

### 1.2.2 宣告式語言（Declarative Language）

宣告式語言描述「做什麼」（What to compute）而非「如何做」（How to compute）。程式設計師只宣告期望的結果，由語言的實作自動推導出計算過程。

**宣告式語言的分類：**

```
宣告式語言
    ├── 函數式語言 (Haskell, LISP, Erlang)
    ├── 邏輯式語言 (Prolog, Datalog)
    ├── 資料查詢語言 (SQL, XQuery)
    └── 組態語言 (JSON, YAML, XML)
```

**SQL - 宣告式資料查詢**

SQL 是最具影響力的宣告式語言之一，用於關聯式資料庫操作：

```sql
-- 從員工表中選取薪水超過 50000 的員工姓名
SELECT name FROM employees WHERE salary > 50000;
```

這段程式碼只描述了「我要的結果」——薪水超過 50000 的員工姓名。SQL 引擎負責決定：
- 如何掃描資料表
- 使用哪個索引
- 以什麼順序處理條件

**宣告式的優點：**

1. **可讀性高**：近似自然語言描述
2. **可最佳化**：實作可自由選擇最佳演算法
3. **宣告式推理**：易於數學驗證
4. **並行友好**：無隱藏副作用，易於並行化

### 1.2.3 函數式語言（Functional Language）

函數式語言以數學函數為核心概念，強調：
- **不可變性**：資料一旦建立不可修改
- **一等函數**：函數可作為參數傳遞和返回值
- **無副作用**：函數呼叫不回應影響全域狀態

**λ 演算法（Lambda Calculus）**

函數式語言的理論基礎是 Alonzo Church 發明的 λ 演算法。λ 演算法只有三個建構：

1. **變數**：`x`
2. **函數抽象**：`λx.e`（表示輸入 x 產生 e）
3. **函數應用**：`(f x)`（將 f 應用於 x）

**LISP 範例**

LISP（LISt Processing）是第二古老的高階語言，其核心思想是「程式即資料」：

```lisp
;; LISP - 計算 1 到 10 的總和
(defun sum (n)
  (if (= n 0)
      0
      (+ n (sum (- n 1)))))

(print (sum 10))  ; 輸出 55
```

LISP 使用「S-表達式」（S-Expression）作為語法，所有程式碼和資料都是列表結構。

**Haskell 範例**

Haskell 是純函數式語言的代表，所有函數都是純函數（無副作用）：

```haskell
-- Haskell - 使用遞迴與模式匹配
sum' :: [Int] -> Int
sum' [] = 0                    -- 空列表基底案例
sum' (x:xs) = x + sum' xs     -- 首元素 + 尾部的總和

main = print (sum' [1..10])    -- 輸出 55
```

**函數式語言的優勢：**

| 特性 | 優勢 |
|------|------|
| 不可變性 | 無競爭條件，易於並行 |
| 純函數 | 易于測試和推理 |
| 高階函數 | 程式碼簡潔、可重用 |
| 延遲求值 | 處理無限資料結構 |
| 模式匹配 | 表達力強、減少錯誤 |

### 1.2.4 邏輯式語言（Logic Language）

邏輯式語言基於一階邏輯，透過宣告事實和規則，讓系統自動進行邏輯推論。

**Prolog 範例**

Prolog（Programming in Logic）的核心概念：

```prolog
% 定義事實
father(tom, bob).
father(tom, alice).
mother(mary, bob).

% 定義規則
parent(X, Y) :- father(X, Y).
parent(X, Y) :- mother(X, Y).

% 查詢：誰是 Bob 的父母？
% ?- parent(Who, bob).
% Who = tom ;
% Who = mary
```

Prolog 的執行模型：
1. **事實（Facts）**：基本真實陳述
2. **規則（Rules）**：邏輯蘊含關係
3. **查詢（Queries）**：提問系統回答

**搜尋策略**

Prolog 使用回溯搜尋（Backtracking）來滿足查詢：
- 嘗試匹配規則頭部
- 遞迴滿足子目標
- 失敗時回溯嘗試其他選項

### 1.2.5 物件導向語言（Object-Oriented Language）

物件導向語言將資料和操作封裝為「物件」，透過訊息傳遞實現物件間的互動。

**OOP 的核心概念：**

| 概念 | 說明 | 範例 |
|------|------|------|
| 封裝（Encapsulation） | 將資料和操作結合成單元 | 類別的成員變數和成員函數 |
| 繼承（Inheritance） | 建立類別階層，共享特性 | Animal → Dog, Cat |
| 多型（Polymorphism） | 不同物件對同一訊息有不同響應 | `draw()` 方法在不同圖形有不同的實作 |
| 抽象（Abstraction） | 隱藏複雜性，只暴露介面 | 介面和抽象類別 |

**Smalltalk - 第一個純物件導向語言**

Smalltalk（1972-1980）奠定了現代 OOP 的基礎，所有東西都是物件，包括類別本身。

```smalltalk
"Smalltalk 範例"
| rect |
rect := Rectangle new.
rect width: 5; height: 3.
rect area print
```

**Python 的 OOP 範例：**

```python
# Python - 物件導向範例
class Rectangle:
    def __init__(self, width, height):
        self.width = width
        self.height = height
    
    def area(self):
        return self.width * self.height

rect = Rectangle(5, 3)
print(rect.area())  # 輸出 15
```

## 1.3 語法與語意

### 1.3.1 語法（Syntax）的定義

語法定義了語言中合法的表示方式——哪些字元序列構成有效的程式。語法規則通常用上下文無關文法（Context-Free Grammar）描述。

**語法的層次：**

```
詞法層次（Lexical Structure）
    - 關鍵字、識別字、運算子、常數
    - 正規表達式描述

語法層次（Syntactic Structure）
    - 陳述式、運算式、宣告
    - 文法規則描述

語意層次（Semantic Structure）
    - 類型檢查、作用域解析
    - 動作語意、表示語意
```

**相同語法，不同語意**

相同的語法結構在不同語言中有完全不同的語意：

```c
// C 語言：= 是賦值運算子
int x = 5;
int y = x = 10;  // x 被設為 10，y 也被設為 10（賦值返回賦值後的值）
```

```python
# Python：== 是相等比較，= 是賦值
x = 5
y = (x == 10)  # y 為 False
```

```javascript
// JavaScript：== 會做型別轉換，=== 嚴格相等
console.log(5 == "5");   // true（字串轉為數字）
console.log(5 === "5");  // false（型別不同）
```

### 1.3.2 語意（Semantics）的類型

語意定義了程式的意義——當程式執行時會發生什麼。

**語意的分類：**

| 類型 | 說明 | 分析方法 |
|------|------|----------|
| 操作語意（Operational） | 定義抽象機器的行為 | 直譯器追蹤狀態 |
| 表示語意（Denotational） | 將語法對應到數學物件 | 函數映射 |
| 公理語意（Axiomatic） | 定義前後條件 | 霍爾邏輯（Hoare Logic） |
| 指稱語意（Descriptional） | 描述程式行為屬性 | 型別系統、行為類型 |

**操作語意範例**

操作語意將程式解釋為「抽象機器上的狀態轉換」。例如：

```
狀態：{x → 5, y → 0}
陳述：y = x + 1
───────────────────
新狀態：{x → 5, y → 6}
```

**公理語意範例**

公理語意使用「前置條件」和「後置條件」描述陳述式的效果：

```
{ x ≥ 0 }           ← 前置條件
y = sqrt(x);        ← 陳述式
{ y × y = x }       ← 後置條件
```

### 1.3.3 型別系統（Type System）

型別系統是語意的重要組成部分，定義了值的種類以及操作的有效性。

**型別檢查的時機：**

| 時機 | 名稱 | 代表語言 | 優點 |
|------|------|----------|------|
| 編譯時 | 靜態檢查 | C, C++, Java | 早期錯誤偵測 |
| 執行時 | 動態檢查 | Python, JavaScript | 彈性、靈活性 |
| 兩者 | 漸進式 | TypeScript, Python | 兩者兼顧 |

**型別推論**

現代語言支援型別推論——由編譯器自動推斷變數型別：

```rust
// Rust - 型別推論
let x = 5;        // 推斷為 i32
let y = 3.14;     // 推斷為 f64
let z = x as f64 + y;  // 明確轉換
```

## 1.4 語言翻譯簡介

### 1.4.1 語言翻譯的類型

將高階語言轉換為機器可執行的形式主要有三種策略：

```
原始碼
   │
   ├─→ 編譯器 (Compiler) ──→ 目的碼 (機器碼/位元組碼)
   │
   ├─→ 直譯器 (Interpreter) ──→ 直接執行
   │
   └─→ 混合式 (JIT/AOT) ──→ 根據情況編譯或直譯
```

**編譯器（Compiler）**

編譯器在執行前將整個程式翻譯為目標形式：

```
原始碼 → [詞法分析] → [語法分析] → [語意分析] → [最佳化] → [目的碼]
```

優點：執行速度快、不需要原始碼
缺點：編譯時間長、平台相依性

**直譯器（Interpreter）**

直譯器逐句讀取並立即執行原始碼：

```
原始碼 → [讀取] → [解譯] → [執行] → [讀取下一句]...
```

優點：跨平台、互動式、開發快速
缺點：執行速度慢

**混合式翻譯**

現代語言常結合兩種方法：

| 方式 | 說明 | 代表 |
|------|------|------|
| VM 直譯 | 編譯為虛擬機位元組碼後直譯 | Java (JVM), C# (.NET) |
| JIT 編譯 | 執行時熱點程式碼編譯為機器碼 | JavaScript V8, HotSpot |
| AOT 編譯 | 提前編譯為機器碼 | Go, Rust, C++ |

### 1.4.2 編譯流程詳解

**編譯器架構**

現代編譯器分為前端和後端：

```
前端（與語言相關）
原始碼 → 詞法分析 → 語法分析 → 語意分析 → 中間表示 (IR)
                                                          │
                                                          ▼
後端（與硬體相關）                              中間表示 (IR)
                                                          │
                                                          ▼
機器碼 ← 目的碼生成 ← 指令排程 ← 暫存器配置 ← 最佳化處理
```

**詞法分析（Lexical Analysis）**

將字元流轉換為 Token 序列：

```
"int x = 42;" 
        │
        ▼ [詞法分析器]
        │
        ▼
[(KEYWORD, "int"), (IDENT, "x"), (ASSIGN, "="), (NUMBER, 42), (SEMI, ";")]
```

**語法分析（Syntax Analysis）**

將 Token 序列組織為語法樹（Parse Tree）或抽象語法樹（AST）：

```
      宣告
     /    \
  類型    賦值
   │      /  \
 "int"  識別字  常數
          │      │
         "x"    42
```

**語意分析（Semantic Analysis）**

進行類型檢查、作用域解析等：

- 識別字是否已宣告？
- 運算元型別是否相容？
- 函數呼叫參數是否正確？

**最佳化（Optimization）**

在不改變語意的前提下改進程式：

- 常數折疊：`2 + 3` → `5`
- 公共子表達式消除：避免重複計算
- 無用程式碼刪除
- 內聯展開：將函數呼叫展開為函數主體

**目的碼生成（Code Generation）**

將最佳化的 IR 轉換為目標機器的機器碼：

```assembly
# 對應 "x = 42" 的 x86-64 組合語言
movl    $42, -4(%rbp)    # 將 42 存入堆疊上的 x
```

### 1.4.3 實作：C 語言編譯流程

使用 GCC 編譯 C 程式：

```bash
# 編譯並執行
$ gcc hello.c -o hello
$ ./hello
Hello!
```

**編譯過程的各個階段：**

```bash
# 預處理（Preprocessing）
gcc -E hello.c -o hello.i

# 編譯為組語（Compilation）
gcc -S hello.i -o hello.s

# 組譯（Assembly）
gcc -c hello.s -o hello.o

# 連結（Linking）
gcc hello.o -o hello
```

### 1.4.4 Java 的混合式執行

Java 先編譯為位元組碼，再由 JVM 執行：

```bash
# 編譯為位元組碼 (.class)
$ javac Hello.java

# 由 JVM 解譯執行
$ java Hello
Hello!
```

**位元組碼是 JVM 的虛擬機指令集：**

```bash
$ javap -c Hello
Compiled from "Hello.java"
public class Hello {
  public Hello();
    Code:
       0: aload_0
       1: invokespecial #1
       5: return

  public static void main(java.lang.String[]);
    Code:
       0: getstatic #2
       3: ldc #3
       5: invokevirtual #4
       8: return
}
```

這些位元組碼指令（如 `aload_0`、`invokespecial`）由 JVM 直譯執行或 JIT 編譯後執行。

### 1.4.5 翻譯策略的選擇

選擇何種翻譯策略需要權衡多個因素：

| 考量因素 | 編譯 | 直譯 | 混合式 |
|----------|------|------|--------|
| 執行速度 | 快 | 慢 | 中→快 |
| 啟動時間 | 快 | 慢（逐句） | 中 |
| 記憶體 | 少 | 多 | 中 |
| 跨平台 | 需重新編譯 | 原生跨平台 | 可跨平台 |
| 互動性 | 低 | 高 | 高 |
| 複雜度 | 高 | 中 | 高 |

**典型應用場景：**

- **編譯型**：C、C++、Go、Rust → 系統程式、作業系統、遊戲引擎
- **直譯型**：Python、Ruby、JavaScript → 腳本、網頁、資料處理
- **混合型**：Java、C#、Kotlin → 企業應用、Android、跨平台服務
