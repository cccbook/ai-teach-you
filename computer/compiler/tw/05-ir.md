# 5. LLVM IR 語法

## 5.1 什麼是 LLVM IR

LLVM IR（Intermediate Representation）是 LLVM 編譯器架構的核心，是一種與目標機器無關的中間表示形式。LLVM IR 設計為三種等價的形式：
- 記憶體中的資料結構
- 位元碼（.bc 檔案）
- 可讀的文字格式（.ll 檔案）

## 5.2 LLVM IR 基本語法

### 模組（Module）

每個 LLVM IR 檔案是一個模組，包含目標描述、全域變數和函數定義：

```llvm
; ModuleID = 'example.c'
source_filename = "example.c"
target datalayout = "e-m:o-i64:64-f80:128-n8:16:32:64-S128"
target triple = "x86_64-apple-darwin"
```

### 類型系統

LLVM IR 提供基本的類型：

| IR 類型 | C 對應 | 說明 |
|--------|-------|------|
| `i1` | `bool` | 1 位元整數 |
| `i8` | `char` | 8 位元整數 |
| `i32` | `int` | 32 位元整數 |
| `i64` | `long` | 64 位元整數 |
| `float` | `float` | 32 位元浮點數 |
| `double` | `double` | 64 位元浮點數 |
| `ptr` | `void*` | 指標類型 |

### 全域變數

```llvm
@global_var = global i32 42
@str = constant [13 x i8] c"Hello World\00"
```

## 5.3 函數定義

[程式檔案：05-1-add.c](../_code/05/05-1-add.c)
```c
int add(int a, int b) {
    return a + b;
}
```

產生的 LLVM IR（未優化）：

[程式檔案：05-1-add.ll](../_code/05/05-1-add.ll)
```llvm
define i32 @add(i32, i32) #0 {
  %a = alloca i32, align 4
  %b = alloca i32, align 4
  store i32 %0, i32* %a, align 4
  store i32 %1, i32* %b, align 4
  %5 = load i32, i32* %a, align 4
  %6 = load i32, i32* %b, align 4
  %7 = add nsw i32 %5, %6
  store i32 %7, i32* %3, align 4
  %8 = load i32, i32* %3, align 4
  ret i32 %8
}
```

使用 `-O2` 優化後的 IR：

[程式檔案：05-1-add-optimized.ll](../_code/05/05-1-add-optimized.ll)
```llvm
define i32 @add(i32, i32) #0 {
  %3 = add nsw i32 %0, %1
  ret i32 %3
}

define i32 @main() #0 {
  ret i32 30
}
```

## 5.4 控制流

### 條件分支（if-else）

[程式檔案：05-2-if.c](../_code/05/05-2-if.c)
```c
int max(int a, int b) {
    if (a > b) {
        return a;
    } else {
        return b;
    }
}
```

產生的 LLVM IR 使用 `icmp` 進行比較，`br` 進行分支，`phi` 合併控制流：

[程式檔案：05-2-if.ll](../_code/05/05-2-if.ll)
```llvm
define i32 @max(i32, i32) #0 {
  %3 = icmp sgt i32 %0, %1
  br i1 %3, label %then, label %else

then:
  ret i32 %0

else:
  ret i32 %1
}
```

### 迴圈（for loop）

[程式檔案：05-3-loop.c](../_code/05/05-3-loop.c)
```c
int sum(int n) {
    int total = 0;
    for (int i = 1; i <= n; i++) {
        total += i;
    }
    return total;
}
```

產生的 LLVM IR：

[程式檔案：05-3-loop.ll](../_code/05/05-3-loop.ll)
```llvm
define i32 @sum(i32) #0 {
  %total = alloca i32
  %i = alloca i32
  store i32 0, i32* %total
  store i32 1, i32* %i
  br label %loop

loop:
  %cond = load i32, i32* %i
  %cmp = icmp sle i32 %cond, %0
  br i1 %cmp, label %body, label %end

body:
  %t = load i32, i32* %total
  %new = add i32 %t, %cond
  store i32 %new, i32* %total
  br label %inc

inc:
  %i_next = add i32 %cond, 1
  store i32 %i_next, i32* %i
  br label %loop

end:
  %result = load i32, i32* %total
  ret i32 %result
}
```

## 5.5 常用指令

### 算術指令

```llvm
%1 = add i32 %a, %b      ; 加法
%2 = sub i32 %a, %b      ; 減法
%3 = mul i32 %a, %b      ; 乘法
%4 = sdiv i32 %a, %b     ; 有號除法
%5 = srem i32 %a, %b     ; 有號餘數
```

### 邏輯指令

```llvm
%1 = icmp eq i32 %a, %b   ; ==
%2 = icmp ne i32 %a, %b   ; !=
%3 = icmp slt i32 %a, %b  ; <
%4 = icmp sle i32 %a, %b  ; <=
%5 = icmp sgt i32 %a, %b  ; >
%6 = icmp sge i32 %a, %b  ; >=
```

### 記憶體指令

```llvm
%ptr = alloca i32, align 4    ; 配置堆疊空間
store i32 %val, i32* %ptr     ; 儲存值
%loaded = load i32, i32* %ptr  ; 載入值
%elem = getelementptr i32, i32* %arr, i32 %idx  ; 指標運算
```

### 函數指令

```llvm
%result = call i32 @foo(i32 %arg)  ; 函數呼叫
ret i32 %result                     ; 返回
ret void                            ; void 返回
```

## 5.6 使用 clang 產生 LLVM IR

```bash
# 產生未優化的 IR
clang -S -emit-llvm -O0 example.c -o example.ll

# 產生優化的 IR
clang -S -emit-llvm -O2 example.c -o example_opt.ll

# 使用 opt 工具進行特定優化
opt -S -O2 -mem2reg example.ll -o example_opt.ll

# 產生位元碼
clang -c -emit-llvm example.c -o example.bc
```

## 5.7 本章小結

本章介紹了 LLVM IR 的基本語法：
- LLVM IR 是與目標機器無關的中間表示
- 使用三種形式：記憶體、位元碼、文字格式
- 基本類型包括整數、浮點數、指標
- 控制流使用分支和 phi 指令
- 可透過 clang 和 opt 工具產生和優化

## 練習題

4. 實作一個簡單的工具來分析 LLVM IR。
