# 語意分析 (Semantic Analysis)

## 概述

語意分析是編譯器的第三個階段，負責檢查程式碼的語義正確性，確保程式碼有意義。

## 主要任務

### 1. 類型檢查

確保運算元的類型相容：

```c
int a = 10;
float b = 3.14;
int c = a + b;  // 錯誤：類型不相容
```

### 2. 作用域分析

確保識別字在可訪問的範圍內使用：

```c
int x = 10;  // 全域作用域
void foo() {
    int x = 20;  // 本地作用域
}
```

### 3. 符號表管理

維護識別字的相關資訊：

| 欄位 | 說明 |
|-----|------|
| 名稱 | 識別字名稱 |
| 類型 | 資料類型 |
| 作用域 | 所屬作用域 |
| 位址 | 記憶體位址 |
