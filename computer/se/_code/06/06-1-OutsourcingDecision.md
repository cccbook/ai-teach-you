# 外包決策矩陣

> 來源：第 6 章「外包策略」

---

## 自己做 vs 外包

### 自己做

| 項目 | 原因 |
|------|------|
| 核心功能開發 | 智慧財產權、迭代速度 |
| AI Prompt 優化 | 累積Know-how |
| 商業邏輯 | 競爭優勢 |
| 客戶資料處理 | 隱私安全 |
| 日常營運 | 成本控制 |

### 外包

| 項目 | 原因 | 平台 |
|------|------|------|
| UI/UX 設計 | 審美專業要求 | Fiverr, 99designs |
| 標誌設計 | 需要專業創意 | Dribbble |
| 滲透測試 | 需要認證專業 | HackerOne |
| 法律文件 | 需要專業資格 | 律師事務所 |
| 雲端架構 | 需要專業經驗 | 顧問公司 |

---

## 決策檢查清單

- [ ] 這項工作需要專業認證嗎？
- [ ] 這項工作是核心競爭力嗎？
- [ ] 我有時間做好這件事嗎？
- [ ] 外包成本vs時間節省？
- [ ] 智慧財產權歸屬？
