# 14. AI 技術對系統程式領域的影響

人工智慧，特別是深度學習，在過去十年中經歷了爆發式增長，從根本上改變了計算機科學的多個領域。對於系統程式設計而言，AI 的影響體現在多個層面：專用 AI 硬體的興起要求系統軟體進行相應適配；機器學習框架的優化涉及編譯器、執行時和作業系統的深度整合；AI 輔助程式設計工具正在改變軟體開發的工作方式。本章將探討 AI 加速器架構、ML 框架優化、AI 推理引擎部署，以及 AI 對系統程式設計的深遠影響。

## 14.1 AI 加速器與硬體

### 14.1.1 GPU 架構

GPU（Graphics Processing Unit）最初設計用於圖形渲染，但因其大規模並行架構而成為深度學習訓練的主力硬體。理解 GPU 架構對於系統程式設計師最佳化 ML 工作負載至關重要。

**CPU vs GPU 架構哲學**

CPU 和 GPU 代表了兩種不同的運算密集型設計哲學：

```
CPU 架構：
┌────────────────────────────────────────┐
│     Control     │     ALU     │   Cache  │
│    (控制)      │   (運算)    │  (快取)  │
├────────────────────────────────────────┤
│     Control     │     ALU     │   Cache  │
│    (控制)      │   (運算)    │  (快取)  │
├────────────────────────────────────────┤
│         ... (少數核心，大量快取)       │
└────────────────────────────────────────┘

GPU 架構：
┌────────────────────────────────────────┐
│  ALU  │  ALU  │  ALU  │  ALU  │ ALU  │
├────────────────────────────────────────┤
│  ALU  │  ALU  │  ALU  │  ALU  │ ALU  │
├────────────────────────────────────────┤
│  ALU  │  ALU  │  ALU  │  ALU  │ ALU  │
├────────────────────────────────────────┤
│  ALU  │  ALU  │  ALU  │  ALU  │ ALU  │
├────────────────────────────────────────┤
│         Control      │      Cache      │
└────────────────────────────────────────┘
     (少量控制)        (小型快取)
```

| 特性 | CPU | GPU |
|------|-----|-----|
| 設計目標 | 低延遲、單執行緒效能 | 高吞吐、平行運算 |
| 核心數 | 數十個（高效能核心） | 數千個（小型核心） |
| 時脈 | 高（3-5 GHz） | 中等（1-2 GHz） |
| 快取 | 大型（L1/L2/L3） | 小型（共用記憶體） |
| 控制邏輯 | 複雜（分支預測、超純量） | 簡單（SIMT） |
| 記憶體頻寬 | 高延遲但低頻寬 | 低延遲但高頻寬 |

**GPU 硬體架構（以 NVIDIA Ampere 為例）**

```
┌──────────────────────────────────────────────────────────────┐
│                      GPU                                  │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐           │
│  │  GPC 1 │ │  GPC 2 │ │  GPC 3 │ │  GPC 4 │  ...    │
│  │(Graphics│ │(Graphics│ │(Graphics│ │(Graphics│          │
│  │Processing│ │Processing│ │Processing│ │Processing│          │
│  │ Cluster) │ │ Cluster) │ │ Cluster) │ │ Cluster) │          │
│  └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘          │
│       │             │             │             │             │
│       ▼             ▼             ▼             ▼             │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐     │
│  │   SM   │ │   SM   │ │   SM   │ │   SM   │ │   SM   │     │
│  │(Stream │ │(Stream │ │(Stream │ │(Stream │ │(Stream │     │
│  │Multiprocessor)                                       │
└──────────────────────────────────────────────────────────────┘
```

- **GPC（Graphics Processing Cluster）**：GPU 的主要組織單元
- **SM（Streaming Multiprocessor）**：GPU 的基本執行單元
- **SP（Streaming Processor）/CUDA Core**：實際執行運算的單元
- **Tensor Core**：專為矩陣運算設計的硬體單元
- **RT Core**：硬體光線追蹤單元

**CUDA 執行模型**

CUDA 將運算組織為三層結構：

| 概念 | 說明 |
|------|------|
| Thread（執行緒） | 最基本執行單位 |
| Thread Block（執行緒區塊） | 一組共享資源的執行緒，最大 1024 個 |
| Grid（網格） | 所有執行緒區塊的集合 |

```
Grid
├── Thread Block 0
│   ├── Thread 0
│   ├── Thread 1
│   └── ...
├── Thread Block 1
│   ├── Thread 0
│   ├── Thread 1
│   └── ...
└── Thread Block N
```

CUDA 執行模型稱為 SIMT（Single Instruction Multiple Thread）， warp 中的 32 個執行緒執行相同指令但處理不同資料。

### 14.1.2 TPU 與專用 AI 加速器

TPU（Tensor Processing Unit）是 Google 為深度學習工作負載設計的專用積體電路（ASIC），代表了一種與 GPU 不同的設計哲學——針對特定工作負載進行硬體最佳化。

**TPU 架構**

```
┌────────────────────────────────────────────────────────────┐
│                      Host Interface                        │
│                    (PCIe/NCI)                             │
└─────────────────────────────┬──────────────────────────────┘
                              │
                              ▼
┌────────────────────────────────────────────────────────────┐
│                    Unified Buffer (UB)                      │
│                   (24MB on-chip SRAM)                      │
└─────────────────────────────┬──────────────────────────────┘
                              │
                              ▼
┌────────────────────────────────────────────────────────────┐
│                      Matrix Multiply Unit                    │
│                   (Systolic Array: 128x128)                │
└─────────────────────────────┬──────────────────────────────┘
                              │
                              ▼
┌────────────────────────────────────────────────────────────┐
│                   Accumulators / Activation                │
│                    (ReLU, Sigmoid, etc.)                  │
└────────────────────────────────────────────────────────────┘
```

**脈動陣列（Systolic Array）**

TPU 的核心是脈動陣列，這是一種專門為矩陣乘法設計的硬體結構：

- 資料沿著陣列「流動」，每個處理單元（PE）執行乘加運算
- 權重預先載入，輸入資料流過陣列
- 這種設計實現了極高的運算密度和能源效率

**TPU vs GPU**

| 特性 | TPU | GPU |
|------|-----|-----|
| 設計哲學 | 專用（矩陣乘法為核心） | 通用（圖形+運算） |
| 精度支援 | INT8, BF16, FP32 | FP16, BF16, FP32, FP64 |
| 記憶體 | 專用 HBM | GDDR/HBM |
| 彈性 | 低 | 高 |
| 最適合 | 批量推理、訓練 | 研究、多樣工作負載 |

### 14.2.2 模型最佳化技術

將大型模型部署到資源受限的環境需要多種優化技術：

**量化（Quantization）**

[_code/14/14_01_quantization.c](_code/14/14_01_quantization.c)

```c
#include <stdio.h>

void quantize(float *input, int8_t *output, int size, float scale) {
    for (int i = 0; i < size; i++) {
        float val = input[i] / scale;
        if (val > 127) val = 127;
        if (val < -128) val = -128;
        output[i] = (int8_t)(val + 0.5f);
    }
}

int main() {
    float input[] = {1.0f, 2.0f, 3.0f, 4.0f};
    int8_t output[4];
    
    quantize(input, output, 4, 0.1f);
    
    printf("Quantized: ");
    for (int i = 0; i < 4; i++) printf("%d ", output[i]);
    printf("\n");
    
    return 0;
}
```

**剪枝（Pruning）**

[_code/14/14_02_pruning.c](_code/14/14_02_pruning.c)

```c
#include <stdio.h>

void prune(float *weights, int size, float threshold) {
    int pruned = 0;
    for (int i = 0; i < size; i++) {
        if (weights[i] > -threshold && weights[i] < threshold) {
            weights[i] = 0.0f;
            pruned++;
        }
    }
    printf("Pruned: %d/%d (%.1f%%)\n", pruned, size, 100.0f * pruned / size);
}

int main() {
    float weights[] = {0.1f, -0.05f, 0.8f, 0.3f, -0.2f, 0.9f};
    int size = 6;
    
    printf("Before pruning: ");
    for (int i = 0; i < size; i++) printf("%.2f ", weights[i]);
    printf("\n");
    
    prune(weights, size, 0.25f);
    
    printf("After pruning: ");
    for (int i = 0; i < size; i++) printf("%.2f ", weights[i]);
    printf("\n");
    
    return 0;
}
```

**知識蒸餾（Knowledge Distillation）**

知識蒸餾使用大型教師模型指導小型學生模型的訓練：

```
┌─────────────────┐
│  Teacher Model   │  Large, High Accuracy
│   (大模型)       │
└────────┬────────┘
         │ 軟目標（機率分佈）
         ▼
┌─────────────────┐
│  Student Model   │  Small, Deployable
│   (小模型)       │
└────────┬────────┘
         │ 蒸餾損失 + 交叉熵
         ▼
    蒸餾損失 = KL(Teacher_soft, Student_soft) * T²
              + CrossEntropy(Student_hard, labels)
```

## 14.4 分散式訓練與系統設計

### 14.4.1 資料平行（Data Parallelism）

資料平行是最常用的分散式訓練策略：

```
┌────────────────────────────────────────────────────────────┐
│                    同步資料平行                              │
├────────────────────────────────────────────────────────────┤
│     GPU 0          GPU 1          GPU 2          GPU 3   │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐  │
│  │ Model    │  │ Model    │  │ Model    │  │ Model    │  │
│  │ Copy     │  │ Copy     │  │ Copy     │  │ Copy     │  │
│  │ (完整)   │  │ (完整)   │  │ (完整)   │  │ (完整)   │  │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘  │
│       ↓              ↓              ↓              ↓      │
│   計算梯度      計算梯度        計算梯度        計算梯度    │
│       ↓              ↓              ↓              ↓      │
│   ┌───────────────────────────────────────────────┐      │
│   │              AllReduce (梯度平均)              │      │
│   └───────────────────────────────────────────────┘      │
│       ↓              ↓              ↓              ↓      │
│   更新權重      更新權重        更新權重        更新權重   │
└────────────────────────────────────────────────────────────┘
```

**AllReduce 演算法**

AllReduce 將多個節點的梯度聚合並廣播結果：

```
Ring AllReduce（減少通訊量）：

Step 1: GPU0→GPU1, GPU2→GPU3 (局部Reduce)
Step 2: GPU1→GPU2, GPU3→GPU0 (繼續Reduce)
Step 3: GPU2→GPU3, GPU0→GPU1 (廣播)
Step 4: GPU3→GPU0, GPU1→GPU2 (廣播)
```

```c
#include <stdio.h>

void allreduce(float *grads, int size, int world_size) {
    for (int i = 0; i < size; i++) {
        float sum = 0;
        for (int r = 0; r < world_size; r++) {
            sum += grads[i];
        }
        grads[i] = sum / world_size;
    }
}

int main() {
    float grads[] = {1.0f, 2.0f, 3.0f, 4.0f};
    int size = 4;
    int world_size = 4;
    
    printf("Before AllReduce: ");
    for (int i = 0; i < size; i++) printf("%.2f ", grads[i]);
    printf("\n");
    
    allreduce(grads, size, world_size);
    
    printf("After AllReduce: ");
    for (int i = 0; i < size; i++) printf("%.2f ", grads[i]);
    printf("\n");
    
    return 0;
}
```

```python
import threading
import queue

class ProducerConsumer:
    def __init__(self, buffer_size=10):
        self.queue = queue.Queue(maxsize=buffer_size)
        self.producer_done = False
    
    def producer(self, items):
        for item in items:
            self.queue.put(item)
            print(f"Produced: {item}")
        self.producer_done = True
    
    def consumer(self):
        while not self.producer_done or not self.queue.empty():
            item = self.queue.get()
            print(f"Consumed: {item}")
            self.queue.task_done()
```

## 14.6 未來趨勢與挑戰

### 14.6.1 硬體趨勢

**記憶體運算（CIM/PIM）**

傳統架構中，資料需要在運算單元和記憶體之間來回傳輸，成為效能瓶頸。記憶體運算將簡單的運算帶入記憶體陣列：

```
傳統架構：
CPU ←─────── 資料匯流排 ────────→ DRAM
         (頻寬受限)

記憶體運算：
┌─────────────────────────────────┐
│           DRAM Array             │
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐      │
│  │PE │ │PE │ │PE │ │PE │  ←─ 記憶體內的處理元素
│  └───┘ └───┘ └───┘ └───┘      │
└─────────────────────────────────┘
    只需傳回結果，資料不需離開記憶體
```

**光學計算**

光學計算使用光子而非電子進行運算，理論上可實現極高頻寬和極低延遲：
