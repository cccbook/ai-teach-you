# 5. Describing Your Website in Conversation

## Why Description Matters

The quality of AI-generated websites highly depends on how you describe your needs. A vague description will produce a vague website; a clear description will produce a website close to your imagination.

```
Vague Description                    Precise Description
────────────────────────────────────────────────────
"I want to make a website"           "I want a black and white
                                     style portfolio website
                                     for a personal photographer"

"Look good"                          "I want minimalist, Nordic style,
                                     with white and light gray as main colors"

"Have some features"                 "I want a large hero image on the homepage,
                                     with three service item cards below"
```

## Basic Framework for Description: 5W1H

When describing website needs to AI, you can use this framework:

| Element | Question | Example Answer |
|---------|----------|---------------|
| **What** | What kind of website? | Personal blog, portfolio, online store |
| **Who** | Who is it for? | Potential clients, employers, fans |
| **Why** | Why is it needed? | Showcase work, sell products, share knowledge |
| **Where** | Where will it be used? | Taiwan market, global Chinese, tech industry |
| **When** | When is it needed? | Launch immediately, one month later |
| **How** | How should it look? | Minimalist, fancy, professional, friendly |

## Description Hierarchy

### Level 1: Website Type

First determine the type of website:

- **Personal Blog**: Share articles, ideas, knowledge
- **Brand Image Website**: Introduce company or personal brand
- **Portfolio Website**: Showcase creative work or services
- **E-commerce Website**: Sell products
- **Landing Page**: Single page promoting specific product or campaign

### Level 2: Core Features

Determine what features the website needs:

- Blog system
- Portfolio showcase (gallery)
- Contact form
- Online booking
- Membership system
- Shopping cart
- Forum/comment board

### Level 3: Visual Style

Describe the visual style you want:

**Color preferences**:
- "Black and white as main"
- "Corporate blue and white"
- "Earth tone colors"
- "Colorful and vibrant"

**Style descriptions**:
- "Minimalism"
- "Modern tech feel"
- "Warm and friendly"
- "Elegant and luxurious"

**Layout preferences**:
- "Lots of white space"
- "Information dense"
- "Card-based layout"

## Practical Conversation Examples

### Example 1: Restaurant Website

**Wrong Example**:
```
Make a restaurant website
```

**Correct Example**:
```
I want to make a website for an Italian restaurant,
targeting young office workers aged 25-40,
style should be quality but not stiff,
using warm colors (beige, brown, wine red),
homepage should have a large menu display area,
need a gallery where you can see photos,
also need online reservation functionality,
finally need address and map.
```

### Example 2: Freelance Designer

**Wrong Example**:
```
Make a designer's portfolio
```

**Correct Example**:
```
I'm a UI/UX designer, want a portfolio website,
mainly for potential clients and HR to see,
want clean and sharp modern style,
black background with white text, accented with some bright color,
homepage should have a short self-introduction and CTA button,
portfolio page should use grid display, each project can be clicked to see details,
need contact form functionality.
```

### Example 3: E-commerce Landing Page

**Wrong Example**:
```
Make a page to sell things
```

**Correct Example**:
```
I want to make a single product landing page for skincare products,
goal is to promote purchases,
need product feature introduction, usage instructions,
customer reviews, FAQ section,
finally need purchase button and limited-time offer countdown,
overall style clean and fresh, using white and light green.
```

## Tips for Describing to AI

### Tip 1: State Conclusion First

Put the most important information at the beginning:

```
Good: "I want a minimalist photographer portfolio, the focus is
on showcasing work, so maximize the photos, minimize text, hide
the navigation, show full content when clicked."

Bad: "Let me tell you first, I'm a photographer, I have lots of
work, I want... (500 words later)... so I need a website to showcase."
```

### Tip 2: Provide References

If you have a website you like, directly provide the link:

```
"I want to make a clean tech product website like Apple,
large images that can slide, very little text but powerful."
```

### Tip 3: List Essential Elements

Use a checklist to ensure important elements aren't missed:

```
Website needs to include:
1. Homepage hero + company slogan
2. Service items (3-4 items)
3. Team member introductions
4. Client logo showcase
5. Contact form
6. Map
```

### Tip 4: Explain What You Don't Want

Sometimes explaining "what you don't want" is also important:

```
"I want a yoga studio website,
want warm and natural feeling,
don't use too much purple (that's the competitor's color),
don't use flash animations, must work on mobile."
```

### Tip 5: Describe in Layers

First give the general direction, then add details:

```
First round: "I want an influencer e-commerce website"
↓
Second round: "Clothing and accessories as main, style should be fashionable, young"
↓
Third round: "Specifically, need new arrival section, bestsellers section,
live streaming section, each product page should have size chart, outfit recommendations"
```

## Use AI to Help You Describe

You can also ask AI to help organize your thoughts first:

### Ask ChatGPT

```
"I want to make a XXX type of website,
but I don't know how to describe my needs,
can you ask me some questions to help organize?"
```

AI will guide you through a series of questions to help you systematically think about your needs.

### Let AI Organize for You

Throw your vague idea to AI:

```
"I have a rough idea: I want to make a website that helps
new parents learn parenting knowledge, but I'm not sure
exactly how to do it. Can you help organize the possible
features and structure?"
```

## Iteration Process After Description

What if the results don't meet expectations after AI generation?

### Step 1: Point Out Problems

Don't just say "not good," be specific:

```
"Problems this time:
1. Colors are too dark, need to be brighter
2. Font is too small, elderly people can't see clearly
3. Want logo on the left, not centered"
```

### Step 2: Don't Change Too Much at Once

Modify 2-3 places at a time, too many changes AI might miss or misunderstand.

### Step 3: Give Positive Feedback

If AI did something right, say thank you before suggesting changes:

```
"Great! The hero image effect is very good.
Small adjustment: 1. Change background color from dark gray to white
2. Move phone number to footer"
```

## Common Description Sentence Patterns

Remember these common patterns for smoother descriptions:

### Feature Types
- "Need... function"
- "Must have... for it to work"
- "Can skip... but need at least..."

### Style Types
- "Style should be like..."
- "Don't want... feeling"
- "Overall atmosphere is..."

### Layout Types
- "...should be on top/bottom/left/right"
- "...should be bigger/smaller"
- "...put... in middle, ... on side"

### Priority Types
- "Most important is..."
- "...can skip but... must have"
- "If can only choose one feature, want..."
