# 2. AI 能做什麼、不能做什麼

## 理解 AI 的邊界

盲目相信 AI 或完全否定 AI，都是新手常犯的錯誤。要成為高效的人機協作者，你必須清楚了解 AI 的能力邊界。

## AI 擅長的事

### 1. 程式碼生成
AI 能夠根據描述快速生成符合語法的代碼。這涵蓋：
- 標準演算法的實現
- 常見設計模式的應用
- CRUD 操作
- API 客戶端代碼
- 測試用例

### 2. 語法與文檔查詢
忘記某個函數的參數順序？想知道某個語法的精確用法？AI 是最好的即時文檔。

### 3. 代碼解釋
給 AI 一段你不熟悉的代碼，它能夠解釋：
- 這段代碼在做什麼
- 為什麼這樣設計
- 可能的執行結果

### 4. 重構建議
AI 能夠識別代碼異味（code smells）並提供重構建議。

### 5. 多語言翻譯
將代碼從一種語言翻譯到另一種語言，AI 通常做得很好。

## AI 不擅長的事

### 1. 完全正確的業務邏輯
AI 不了解你的公司業務、產業規則或特殊需求。它可能生成看似合理但邏輯錯誤的代碼。

```
問題示例：
你：「為用戶創建一個訂單」
AI 可能：「就這樣創建訂單」
你需要的：「檢查庫存、計算折扣、驗證優惠券、發送通知、更新會員積分...」
```

### 2. 完整系統的設計
AI 適合幫你設計單一組件，但難以把握整個系統的複雜度。一個涉及 10 個微服務、5 個資料庫的系統，需要人類來確保架構的一致性和可行性。

### 3. 最新技術和工具
AI 的知識有截止日期。它可能不知道：
- 上週發布的新框架
- 昨天修復的安全漏洞
- 明天即將廢除的 API

### 4. 沒有標準答案的問題
「這個架構好不好？」「這個設計是否合理？」這類問題需要業務上下文和人類判斷。

### 5. 處理意外的邊界情況
生產環境的問題往往出人意料。AI 擅長處理常見問題，但面對從未見過的錯誤時能力有限。

## 實用原則：紅旗清單

以下情況，**不要**依賴 AI 的直接答案：

| 情境 | 原因 | 正確做法 |
|------|------|----------|
| 安全相關的代碼 | AI 可能產生漏洞 | 人工審查 + 安全工具掃描 |
| 需要登入授權的 API | AI 沒有你的憑證 | 查看官方文檔 |
| 最新發布的框架 | 知識可能過時 | 查看官方教程 |
| 涉及敏感資料 | 資訊安全風險 | 本地處理 |
| 法律/合規問題 | 需要專業判斷 | 諮詢專業人士 |

## 紅燈思維

培養這個習慣：每次看到 AI 的回應時，問自己：

```
1. 這個回答合理嗎？
2. 有沒有我沒考慮到的情況？
3. 如果這個代碼執行，會發生什麼？
4. 我需要什麼測試來驗證？
```

## 最佳實踐：配對程式設計模式

將 AI 視為你的「見習工程師」：

```
┌─────────────┐         ┌─────────────┐
│   人類      │  ←────→  │     AI      │
│  (資深工程師)│  協作    │ (見習工程師) │
└─────────────┘         └─────────────┘

人類職責：審查、決策、把關
AI 職責：生成、查詢、嘗試
```

你仍然是最終負責的人，AI 只是你的工具。
