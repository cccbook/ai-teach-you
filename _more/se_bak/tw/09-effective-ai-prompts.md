# 9. 如何有效地向 AI 提問

## 提問是核心技能

「問得好，才能答得好。」這句話在 AI 時代比以往任何時候都重要。

## 為什麼同樣的 AI，不同的結果

```
两个工程师用同一个 AI：

工程师 A：
「我的代码报错了」
AI：「无法定位问题，需要更多信息」

工程师 B：
「我的代码报错了，错误信息是 'TypeError: Cannot read property 
 of undefined at line 23'，这是相关代码：
[代码片段]'
AI：「问题在于第 18 行返回的对象没有 'user' 属性...
```

**工程师 B 得到了有用的答案，因为他的提问更有信息量。**

## 提問的層次

### 第一層：模糊問題

```
❌ 不好：
「怎么做一个网站？」
「代码报错怎么办？」
「Python 怎么学？」
```

AI 能給你的是泛泛的回答，通常是通用建議或官方文檔的摘要。

### 第二層：具體問題

```
✅ 还好：
「怎么用 React 实现用户登录功能？」
「Python 中 list 和 tuple 的区别是什么？」
```

AI 能給你實用的答案，但可能不是最優解。

### 第三層：上下文完整的問題

```
✅✅ 最好：
「我正在用 React + TypeScript 开发一个博客系统，
需要实现第三方登录（Google）。
目前用的是 Firebase Auth，但遇到 CORS 错误。
已经尝试过的方法：[具体方法]
错误截图：[描述]
请问如何解决？」
```

AI 能給你針對性的解決方案。

## 有效提問的要素

### 要素一：上下文（Context）

```
❌ 不好：
「Redis 连接报错」

✅ 好：
「我的 Node.js 应用在 Kubernetes 环境中使用 Redis，
在本地测试正常，但在生产环境每秒请求超过 100 时出现
'ECONNREFUSED' 错误」
```

### 要素二：目標（Goal）

```
❌ 不好：
「这个代码怎么优化？」

✅ 好：
「这个代码在处理 100 万条数据时需要 30 秒，
需要优化到 5 秒以内」
```

### 要素三：約束（Constraints）

```
❌ 不好：
「帮我写一个排序算法」

✅ 好：
「帮我写一个排序算法，要求：
1. 在大部分已有序的数据上表现好
2. 空间复杂度 O(1)
3. 用 JavaScript 实现」
```

### 要素四：已嘗試的方法（Attempts）

```
❌ 不好：
「Git 合并冲突怎么解决？」

✅ 好：
「我在解决 Git 合并冲突时，用了 'git merge --abort' 
取消合并，但再次合并时仍然报错，
错误是 'refusing to merge unrelated histories'，
请问我应该怎么处理？」
```

## 提問模板

### 模板一：代碼除錯

```
## 问题描述
[简要描述遇到的问题]

## 环境信息
- 操作系统：
- 编程语言及版本：
- 相关框架/库版本：
- 运行环境（本地/服务器/容器）：

## 错误信息
```
[完整的错误信息]
```

## 相关代码
```[语言]
[代码片段]
```

## 已尝试的解决方法
1. ...
2. ...

## 预期行为
[描述你期望的结果]
```

### 模板二：功能實現

```
## 需求描述
[简要描述你想要实现的功能]

## 背景
[为什么需要这个功能]

## 技术约束
- 编程语言：
- 框架版本：
- 数据库（如适用）：
- 其他限制：

## 参考资料
[相关的文档、教程、代码片段]

## 具体问题
[你特别想了解的部分]
```

### 模板三：代碼審查

```
## 审查目标
[描述这段代码的目的]

## 代码
```[语言]
[代码]
```

## 我的顾虑
[你特别想了解的部分]
1. ...
2. ...

## 代码风格要求
[如有特定的代码风格要求]
```

## 常見提問錯誤

### 錯誤一：要求太多

```
❌ 不好：
「帮我做一个抖音那样的 APP」

✅ 好（分步提问）：
1. 「帮我设计一个短视频 APP 的技术架构」
2. 「移动端视频上传的技术方案有哪些？」
3. 「帮我设计视频推荐系统的数据库模型」
```

### 錯誤二：隱藏關鍵資訊

```
❌ 不好：
「这个 Python 代码运行报错」

✅ 好：
「这个 Python 3.9 代码在 Windows 上运行正常，
但在 macOS 上报错 'ModuleNotFoundError'」
（暴露了操作系统差异这个关键信息）
```

### 錯誤三：假設答案

```
❌ 不好：
「这个性能问题是因为数据库没有索引吗？」

✅ 好：
「这个查询很慢，请问可能的原因有哪些？」
（让 AI 全面分析，避免先入为主）
```

### 錯誤四：同時問太多問題

```
❌ 不好：
「什么是微服务？Docker 怎么用？K8s 怎么部署？
怎么实现 CI/CD？」

✅ 好：
「请解释微服务架构的核心概念」
（一个一个问题来）
```

## 迭代式提問

有時候一次提問無法得到完整答案，這是正常的：

```
第一轮：
你：「帮我实现用户登录功能」
AI：[给了一个基本实现]

第二轮：
你：「现在需要支持 Remember Me 功能，保持登录状态 30 天」

第三轮：
你：「还需要在多个设备登录时，能查看并管理登录设备」

第四轮：
你：「最后加上异地登录的安全提醒功能」
```

每次迭代都比上一次更完善。

## 提問檢查清單

在發送問題前，問自己：

```
□ 我是否提供了足够的上下文？
□ 我的问题是否具体明确？
□ 我是否说明了约束条件？
□ 我是否说明了已尝试的方法？
□ 这个问题是否太复杂，需要拆分？
□ 我是否假设了答案的方向？
□ 我是否一次问了太多问题？
```

## 實踐練習

用以下步驟練習提問：

1. 選擇你最近遇到的一個技術問題
2. 先用「模板一」寫下問題
3. 用這個問題問 AI
4. 評估回答的質量
5. 調整問題，再次提問
6. 總結這次經驗

**好問題是成功的一半。投資在提問上，回報是更好的答案和更高的效率。**
