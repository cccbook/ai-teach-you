# 1. From High-Level Language to Machine Code——The Essence of C

## 1.1 Why Learn C Language?

In an era dominated by Python, JavaScript, Go, and Rust, why should we still learn C? The answer is simple: **C is the high-level language closest to hardware, and the best starting point for understanding computer systems.**

When you write this code:

```c
int main() {
    int a = 10;
    int b = 20;
    int c = a + b;
    return c;
}
```

Do you think the computer directly "understands" this code? Wrong. Computers only understand 0 and 1. From your source code to actual machine instructions, there's a complex transformation process.

## 1.2 Compilation Flow Overview

```
source.c (Source Code)
    │
    ▼  [Preprocessor cpp]
Preprocessed Code
    │
    ▼  [Compiler gcc/clang]
Assembly (assembly.s)
    │
    ▼  [Assembler as]
Object File (object.o)
    │
    ▼  [Linker ld]
Executable
```

Let's see this process with a real example. Given `hello.c`:

```c
#include <stdio.h>

int main() {
    printf("Hello, World!\n");
    return 0;
}
```

View each stage with the compiler:

```bash
# Preprocess
gcc -E hello.c -o hello.i

# Compile to assembly
gcc -S hello.c -o hello.s

# Assemble to object file
gcc -c hello.c -o hello.o

# Link to executable
gcc hello.c -o hello
```

## 1.3 What Does the Preprocessor Do?

The preprocessor handles `#include`, `#define`, macro expansion, etc. Look at the beginning of `hello.i` - you'll see the standard header files expanded. The key `#include` expansion inserts stdio.h content into your code. These expanded contents provide the function declarations for `printf`.

## 1.4 What Does the Compiler Do?

The compiler converts C syntax to target platform assembly. Different CPU architectures (x86, ARM, RISC-V) have different assembly languages.

C is called "portable assembly" because:
- No hidden memory management
- No automatic garbage collection
- Pointers let you directly manipulate memory addresses
- Manual stack and heap management

## 1.5 What Does Assembly Look Like?

x86-64 Hello World assembly:

```asm
section .data
    msg db 'Hello, World!', 0
    len equ $ - msg

section .text
global _start
_start:
    mov rax, 1        ; syscall number (write)
    mov rdi, 1        ; file descriptor (stdout)
    mov rsi, msg      ; message pointer
    mov rdx, len      ; message length
    syscall           ; trigger syscall
    mov rax, 60       ; syscall number (exit)
    xor rdi, rdi      ; return value 0
    syscall           ; exit program
```

Equivalent RISC-V:

```asm
.section .data
msg:
    .string "Hello, World!\n"
    len = 14

.section .text
.globl _start
_start:
    li a0, 1          ; file descriptor (stdout)
    la a1, msg        ; message pointer
    li a2, len        ; message length
    li a7, 64         ; syscall number (write)
    ecall
    li a0, 0          ; return value
    li a7, 93         ; syscall number (exit)
    ecall
```

## 1.6 C Language Design Philosophy

Principles from Brian Kernighan and Dennis Ritchie's "The C Programming Language":

1. **Trust the programmer** - C assumes you know what you're doing and won't stop you from doing "dangerous" things
2. **Mechanisms, not policies** - Provide basic mechanisms, let programmers decide how to use them
3. **Simplicity** - Build complex systems from a few but powerful concepts

This design philosophy makes C the preferred language for system programming: operating systems, embedded systems, compilers, databases, drivers.

## 1.7 Secrets of the First C Program

Let's look deeper at the `main` function:

```c
int main(void) {
    // your code
    return 0;
}
```

Legal `main` signatures:
```c
int main(void);           // Standard C
int main(int argc, char *argv[]);  // Accept command line arguments
int main(int argc, char **argv);    // Same as above, different syntax
```

Without a `return`? Since C99, if `main` doesn't explicitly return, it defaults to returning 0.

## 1.8 Compile Your First Program

```bash
# Create directory and file
mkdir -p examples/ch01
cat > examples/ch01/hello.c << 'EOF'
#include <stdio.h>

int main(void) {
    printf("Hello, System Programming!\n");
    return 0;
}
EOF

# Compile and run
gcc -Wall -Wextra -o examples/ch01/hello examples/ch01/hello.c
./examples/ch01/hello
```

`-Wall -Wextra` enables all warnings, helping you write better code.

## 1.9 Summary

In this chapter we learned:
- C is a high-level language close to hardware
- Compilation flow: preprocess → compile → assemble → link
- Assembly is the raw language of the CPU
- C's design philosophy: trust the programmer, simplicity
