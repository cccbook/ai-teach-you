# 39. 作業系統虛擬化——從 xv6 到現代 kernel

## 39.1 虛擬化基礎

虛擬化允許一台硬體同時執行多個作業系統。

```
┌─────────────┐ ┌─────────────┐ ┌─────────────┐
│   Guest 1   │ │   Guest 2   │ │   Guest 3   │
│  (VM 1)    │ │  (VM 2)    │ │  (VM 3)    │
├─────────────┤ ├─────────────┤ ├─────────────┤
│  Guest OS   │ │  Guest OS   │ │  Guest OS   │
├─────────────┤ ├─────────────┤ ├─────────────┤
│  Hypervisor │ (KVM / Xen / VMware)
├─────────────┤
│   Hardware  │ (CPU, Memory, Storage, Network)
└─────────────┘
```

## 39.2 虛擬化類型

| 類型 | 說明 | 例子 |
|------|------|------|
| 全虛擬化 | Guest 無需修改 | VMware, QEMU |
| 半虛擬化 | Guest 需修改 | Xen, VirtIO |
| 硬體輔助 | CPU 支援虛擬化 | KVM, VMware |

## 39.4 從 xv6 到現代 kernel

| 特性 | xv6 | Linux |
|------|-----|-------|
| CPU | 單核心支援 | 多核心支援 |
| 記憶體 | 固定配置 | 動態管理 |
| 檔案系統 | 簡化 v6 | ext4, btrfs |
| 網路 | 無 | 完整 TCP/IP |
| 容器 | 無 | Docker |
