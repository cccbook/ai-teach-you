# 38. 靜態分析——連結器如何解析符號

## 38.1 連結的任務

連結器的主要任務：
1. **符號解析**：找到符號定義
2. **重定位**：修正位址引用
3. **符號綁定**：決定符號最終位址

## 38.2 符號類型

| 類型 | 說明 |
|------|------|
| 強符號 | 函式、全域變數初始化 |
| 弱符號 | 未初始化、全域變數 |
| UND | 未定義（外部引用）|

## 38.3 符號解析規則

1. 多個強符號 → 連結錯誤
2. 一個強符號 + 多個弱符號 → 選擇強符號
3. 多個弱符號 → 選擇任意一個

## 38.4 重定位

### 38.4.1 重定位條目

目的檔包含重定位資訊，告知連結器需要修正的位置：

```bash
readelf -r hello.o
```

## 38.5 常見問題

### 38.5.1 未定義符號

```bash
# 連結錯誤
undefined reference to `foo'
```

解決：加入定義 `foo` 的庫或目標檔。

### 38.5.2 重複定義

```bash
# 連結錯誤
multiple definition of `x'
```

解決：使用 `static` 或 `__attribute__((weak))`。
