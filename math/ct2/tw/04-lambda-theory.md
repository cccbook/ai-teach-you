# 第四章：Lambda Calculus 理論

## 4.1 計算的本質問題

在深入探討Lambda Calculus之前，我們需要回到一個更根本的問題：什麼是計算？這個問題看似簡單，卻困擾了數學家和哲學家很長時間。我們每天都在進行計算——加法、乘法、解決方程——但「計算」這個概念本身卻很難嚴格定義。

「計算」一詞在日常語言中有著廣泛的含義。在最廣義的意義上，計算指的是任何有目的的符號操作過程。例如，用筆算加法是計算，求解微分方程也是計算。然而，這些例子之間存在著巨大的差異。讓我們思考一下：是否存在一個統一的框架，可以描述所有這些看似不同的計算過程？

這個問題在二十世紀初變得特別緊迫。希爾伯特的可判定性問題（Entscheidungsproblem）要求數學家們明確回答「什麼是可計算的」這個問題。可判定性問題問的是：是否存在一個算法，可以判定任意一階邏輯命題的真假？為了回答這個問題，我們首先需要精確地定義什麼是「算法」或「可計算」。

在1936年之前，「可計算」這個概念一直沒有精確的數學定義。數學家常說某個函數是「可計算的」，但這只是一種直觀的說法，依賴於每個人對「計算」的不同理解。哥德爾和赫布蘭德（Herbrand）提出了「遞歸函數」的概念，但這個概念最初並沒有被廣泛接受為「可計算」的最終定義。

丘奇和圖靈的貢獻在於，他們獨立地提出了看似不同但最終等價的計算模型，並大膽地提出論題：這些模型捕捉了「直觀意義下可計算」的全部含義。這就是著名的「丘奇-圖靈論題」。從某種意義上說，丘奇-圖靈論題是一個關於宇宙的形而上學命題：它斷言，任何物理上可實現的計算過程都可以由圖靈機（或Lambda Calculus）模擬。

在本章和後續章節中，我們將詳細探討丘奇發明的Lambda Calculus，這是一個極簡但極其強大的計算模型。通過這個模型，我們可以深入理解計算的本質，並欣賞數學抽象的力量。

## 4.2 Lambda Calculus 的起源與動機

Lambda Calculus（λ演算）是由阿隆佐·丘奇在1930年代初期發明的一種形式系統。丘奇發明Lambda Calculus的初衷是為數學邏輯研究提供一個工具，他試圖用這個系統來表達和推理關於函數的問題。誰也沒有想到，這個看似純粹的數學工具後來會成為計算機科學的理論基石。

丘奇為什麼要發明Lambda Calculus？這要從數學邏輯的發展說起。在1920年代和1930年代，數學家們正在努力解決形式系統的基礎問題。哥德爾的完备性定理（1929年）解決了命題邏輯的完备性問題，但隨後哥德爾的不完备定理（1931年）表明，更強大的系統（如算術）是不完备的。

在這個背景下，丘奇開始思考如何使用函數和函數應用的語言來表達數學命題。函數是數學中最基本的概念之一，但長期以來，函數的精確定義和處理方式一直存在爭議。丘奇試圖創建一個極簡的形式系統，在這個系統中，函數和函數應用是唯一的基本概念。

丘奇在1932年發表了第一篇關於Lambda Calculus的論文，當時他稱之為「論表達式的替換問題」（An Unsolved Problem in the Theory of Effective Calculability）。在這篇論文中，他提出了一種新的函數表示方法，使用希臘字母λ（lambda）來標記函數的參數。這種表示法具有革命性的創新：它允許我們明確地表達「創建函數」這個操作。

例如，在傳統的數學記號中，當我們寫「f(x) = x + 1」時，f是一個函數，但「創建函數f」這個操作本身並沒有被明確地表達出來。在Lambda Calculus中，我們可以寫成「λx. x + 1」，這明確地表達了「這是一個函數，它接受一個參數x，並返回x + 1」。這種表示法使我們能夠將函數視為一類可以被操作和組合的對象。

1936年，丘奇發表了經典論文「論可計算數」，在論文中他提出了使用Lambda Calculus來定義可計算性的想法。同年，他的學生克里尼證明了Lambda Calculus可以計算的函數正好是遞歸函數。這些結果確立了Lambda Calculus作為計算模型的地位。

## 4.3 Lambda Calculus 的語法

Lambda Calculus是一種極簡的形式系統，它的語法非常簡單。我們從一個很小的符號集合開始，定義什麼是合法的「表達式」。

Lambda Calculus的符號表（Alphabet）包括：

- 變量（Variables）：通常用小寫字母表示，如 x, y, z, a, b, c 等。變量代表的是參數位置，它們本身沒有固定的意義。
- 抽象運算符（Abstraction）：λ（lambda）符號，用於創建函數。
- 應用運算符（Application）：沒有顯式的符號，通常通過並置（juxtaposition）表示。
- 括號：用於消歧義。

現在，我們定義什麼是合法的Lambda表達式（Lambda Expression，也稱為λ項）：

**定義（語法）：** Lambda表達式的集合Λ由以下規則生成：

1. 如果x是一個變量，那麼x ∈ Λ。
2. 如果M ∈ Λ且x是一個變量，那麼λx.M ∈ Λ。這個表達式稱為「抽象」（abstraction），表示一個函數，其中x是函數的參數，M是函數體。
3. 如果M, N ∈ Λ，那麼(M N) ∈ Λ。這個表達式稱為「應用」（application），表示將函數M應用於參數N。

這就是Lambda Calculus的全部語法！只有三條簡單的規則，卻可以表達極其豐富的計算內容。

讓我們看一些例子來理解這個語法：

- `x` 是一個合法的表達式，它表示一個變量。
- `λx.x` 是一個合法的表達式，它表示一個函數：接受參數x，並返回x。這個函數稱為「恆等函數」。
- `λx.y` 是一個合法的表達式，它表示一個函數：接受參數x，並返回y。這是一個「常數函數」，它總是返回y，忽略輸入。
- `(λx.x) y` 是一個合法的表達式，它表示將函數λx.x應用於參數y。這個表達式應該簡約為y（因為λx.x是恆等函數）。
- `λx.(x x)` 是一個合法的表達式，它表示一個函數：接受參數x，並將x應用於自身（即計算x(x)）。這個函數在Lambda Calculus中特別重要，它與Y Combinator密切相關。

在書寫時，我們通常省略一些括號，採用以下慣例：

- 應用是左結合的：M N P 意思上是 ((M N) P)。
- 抽象向右延伸：λx.M N 意思上是 λx.(M N)，而不是 (λx.M) N。

這些語法慣例使表達式更容易閱讀。

## 4.4 自由變量與約束變量

在Lambda Calculus中，λ符號有一個重要的作用：它可以「綁定」變量。這與微積分中的定積分變量或程序設計語言中的局部變量類似。我們需要區分「自由變量」和「約束變量」這兩個重要的概念。

**定義（自由變量）：** 一個λ表達式的自由變量（Free Variables）是在表達式中出現但不被任何λ綁定的變量。形式化地，我們可以定義函數FV(-)如下：

1. FV(x) = {x}，其中x是一個變量。
2. FV(λx.M) = FV(M) \ {x}。因為x在λx.M中被綁定了，所以我們從M的自由變量集合中移除x。
3. FV(M N) = FV(M) ∪ FV(N)。

**定義（約束變量）：** 一個λ表達式的約束變量（Bound Variables）是被λ綁定的變量。注意，一個變量可以同時是自由的和約束的，具體取決於它在表達式中的位置。

讓我們看一些例子：

- 表達式 `λx.x` 中，x是被λx綁定的，所以它沒有自由變量。我們可以寫成：FV(λx.x) = ∅。
- 表達式 `λx.y` 中，y不在λ的作用範圍內，所以y是自由變量。FV(λx.y) = {y}。
- 表達式 `(λx.x) y` 中，x是被λ綁定的，y是自由變量。FV((λx.x) y) = {y}。
- 表達式 `λx.(x y)` 中，x被λ綁定，y是自由變量。FV(λx.(x y)) = {y}。

自由變量的概念在Lambda Calculus的语义定义中非常重要。当我们「求值」一个Lambda表达式时，我们只关心它是否会减少（reduce）——我们将在下一节详细讨论这个问题。自由变量的值取决于「环境」，而约束变量则由λ抽象「提供」。

## 4.5 Alpha 轉換

現在我們來討論Lambda Calculus的核心概念：α轉換（Alpha Conversion）。α轉換是關於如何「重命名」約束變量的規則。

在λx.M這個表達式中，變量名x本身其實是任意的。重要的是λ綁定了某個變量，至於這個變量叫什麼名字並不重要。例如，λx.x 和 λy.y 這兩個表達式在直觀上表示的是同一個函數：接受一個參數並返回它們自己。

**定義（α轉換）：** α轉換允許我們將λx.M改寫為λy.M[x/y]，其中M[x/y]表示將M中所有自由出現的x替換為y。但是，為了避免捕捉自由變量，我們要求y不在M的自由變量集合中（即 y ∉ FV(M)）。

這個條件是必要的！讓我們看一個例子：考慮表達式 λx.λy.(x y)。如果我們不假設任何條件就直接將內層的y重命名為x，會得到λx.λx.(x x)，這完全改變了原表達式的含義。原表達式中，外層的x和內層的y是兩個不同的參數，而修改後的表達式中，內層的x實際上被外層的x捕捉了，意義完全不同。這就是「捕捉」（capture）的問題。

**α等價：** 兩個λ表達式如果可以通過有限次α轉換相互轉化，則稱它們是α等價的（α-equivalent）。α等價是Lambda Calculus中的一個重要關係，它表達了「重命名約束變量不改變表達式的意義」這個直觀想法。

在理論上，我們通常將α等價的表達式視為相同的表達式，而不區分它們。也就是說，我們考慮的是「表達式的同餘類」，在這個同餘類中，我們只關心結構，而不關心具體的變量名。這種處理方式稱為「_alpha-等價類_」或「語法範疇」。

## 4.6 Beta 歸約

Beta歸約（β-Reduction）是Lambda Calculus中最核心的概念，它描述了「函數應用」的求值過程。當我們寫(M N)時，我們是將函數M應用於參數N。Beta歸約告訴我們，這個應用如何被「計算」。

**定義（Beta歸約）：** Beta歸約的基本規則是：

(λx.M) N → M[N/x]

這表示：將函數λx.M應用於參數N，相當於將函數體M中的所有自由出現的x替換為N。

讓我們看一些例子：

1. (λx.x) y → y。將恆等函數λx.x應用於y，將x替換為y，得到y。

2. (λx.x x) y → y y。將函數λx.(x x)應用於y，將函數體中的x替換為y，得到(y y)，即將y應用於y。

3. (λx.λy.x) z → λy.z。將接受兩個參數的函數λx.λy.x應用於z，得到λy.z（這裡y不在z中自由出現，所以不需要擔心捕捉問題）。

4. (λx.(x x)) (λx.(x x)) → (λx.(x x)) (λx.(x x))。這個表達式是自己的應用：將函數λx.(x x)應用於自己。這個歸約會無限循環，永遠不會停止。

Beta歸約的替換需要遵循一些規則，以避免捕捉自由變量的問題。替換 M[N/x] 的定義需要递归地处理：當我們要將M中的x替換為N時，如果遇到λy.P，我們需要考慮y是否在N的自由變量中。如果是，我們需要先對M進行α轉換，換一個不衝突的變量名。

Beta歸約是Lambda Calculus中「計算」的核心。我們可以將整個計算過程視為一系列的Beta歸約步驟。當然，我們通常還需要考慮括號和應用的結合方式。

## 4.7 正則序求值與應用序求值

在執行Beta歸約時，我們需要決定「先歸約哪一部分」。有兩種主要的求值策略：正則序求值（Normal Order Evaluation）和應用序求值（Applicative Order Evaluation）。

**正則序求值（又稱最左最外歸約）：** 在正則序求值中，我們總是先歸約「最外層」的函數。具體來說，如果我們有一個應用表達式(M N)，我們首先嘗試歸約M（即最左邊的函數），只有當M已經是「值」（如λ抽象）而無法再歸約時，我們才會去歸約參數N。

正則序求值的一個重要性質是：如果存在任何歸約序列可以使表達式歸約到正則形式（normal form），那麼正則序求值一定能找到這個序列。正則形式是指不能進一步進行Beta歸約的表達式。例如，λx.x是正則形式，因為它是一個λ抽象，不能再被歸約。

**應用序求值（又稱最左最內歸約）：** 在應用序求值中，我們總是先歸約「最內層」的參數。具體來說，如果我們有一個應用表達式(M N)，我們首先嘗試歸約N（即參數），只有當N已經是「值」時，我們才會去歸約M。

應用序求值通常更符合我們對「函數調用」的直觀理解。當我們調用函數時，我們通常先計算所有參數的值，然後再將這些值傳入函數。這也是大多數程序語言（如Python、Java、C）採用的求值策略。

這兩種求值策略有一個重要的區別：有些表達式在應用序求值下可能不會終止，但在正則序求值下會終止。例如，考慮表達式 (λx.y) ((λx.x x) (λx.x x))。這個表達式在正則序求值下會立即歸約為y（因為我們先歸約函數部分，而函數部分忽略參數，直接返回y）。但在應用序求值下，我們會先嘗試歸約參數((λx.x x) (λx.x x))，這個子表達式會無限循環，導致整個計算不終止。

## 4.8 Church-Rosser 定理

Church-Rosser 定理是Lambda Calculus中最重要的定理之一，它表明Beta歸約具有「合流性」（Confluence）。這個定理說：無論我們以什麼順序進行Beta歸約，最終得到的「結果」在α等價意義下都是相同的。

正式地，Church-Rosser定理可以陳述如下：如果一個λ表達式M可以歸約到兩個不同的表達式A和B（即 M →* A 且 M →* B），那麼存在一個表達式C，使得A和B都可以歸約到C（即 A →* C 且 B →* C）。

用通俗的話說，這個性質可以稱為「條條大路通羅馬」：無論我們選擇哪條歸約路徑，只要最終能夠得到一個正則形式（或任何結果），這個結果都是唯一的（在α等價意義下）。

Church-Rosser定理為Lambda Calculus提供了一個重要的理論保障。它告訴我們，雖然一個表達式可能有很多不同的歸約路徑，但這些路徑最終會匯合。我們不必擔心「計算順序」會影響最終結果。當然，計算是否會終止是另一個問題——有些表達式可能會無限循環，永遠達不到正則形式。

這個定理還有一個重要的推論：如果一個λ表達式有正則形式（即可以歸約到一個不能進一步歸約的表達式），那麼這個正則形式是唯一的（准確地說，在α等價意義下是唯一的）。這意味著，當我們說「計算結果」時，我們是在談論唯一確定的對象。

Church-Rosser定理的證明是相當技術性的。我們在這裡只描述其核心思想。證明通常使用「平行歸約」（parallel reduction）的概念，定義一種比普通Beta歸約更「寬鬆」的歸約關係，然後通過歸納證明這種平行歸約具有合流性。

## 4.9 停機問題與不可判定性

在Lambda Calculus的語境下，我們可以討論一個重要的問題：停機問題（Halting Problem）。停機問題問的是：是否存在一個算法，可以判定任意一個λ表達式是否會歸約到正則形式？

答案是否定的。停機問題是不可判定的。這意味著，不存在這樣的算法：給定任何λ表達式M，我們無法確定M是否會終止（歸約到正則形式）。

停機問題的不可判定性可以通過多種方式證明。我們可以將其歸約到圖靈機的停機問題，或者使用對角線方法直接證明。

對角線證明的思路是這樣的：假設存在一個算法H可以判定停機問題。我們構造一個表達式D如下：D ≡ λf.(f f) (λf.(f f))。然後，我們構造另一個表達式X：

X ≡ λg. (如果 g g 能夠終止，則 X X 永遠循環；否則 X X 終止)

現在，問X X是否會終止？如果H說會終止，那麼根據X的定義，X X會進入無限循環；如果H說不會終止，那麼X X會終止。這就導致了矛盾。因此，H不可能存在。

這個結果與圖靈機的停機問題不可判定性密切相關。事實上，我們可以證明：λ表達式是否會終止的問題，與圖靈機是否會停機的問題，是相互歸約的。這進一步支持了丘奇-圖靈論題：不同的計算模型在能力上是等價的。

## 4.10 Lambda Calculus 與遞歸函數

在這一節中，我們要討論Lambda Calculus與遞歸函數（Recursive Functions）之間的深刻聯繫。克里尼（Stephen Kleene）在1936年證明了一個重要定理：Lambda Calculus可以計算的函數正好是遞歸函數。

什麼是遞歸函數？遞歸函數是數學中一類重要的可計算函數，它們使用遞歸定義。遞歸函數的集合由以下三類函數生成：

1. **初始函數**：
   - 零函數：Z(x) = 0
   - 後繼函數：S(x) = x + 1
   - 投影函數：P_i^n(x_1, ..., x_n) = x_i

2. **封閉規則**：
   - **複合**（Composition）：如果f和g_1, ..., g_m是遞歸函數，那麼h(x_1, ..., x_n) = f(g_1(x_1, ..., x_n), ..., g_m(x_1, ..., x_n)) 也是遞歸函數。
   - **原始遞歸**（Primitive Recursion）：如果f和g是遞歸函數，那麼定義為：
     h(x_1, ..., x_n, 0) = f(x_1, ..., x_n)
     h(x_1, ..., x_n, S(y)) = g(x_1, ..., x_n, y, h(x_1, ..., x_n, y))
     的函數h也是遞歸函數。

3. **最小化**（Minimization，又稱μ運算）：如果f是一個遞歸函數，那麼g(x_1, ..., x_n) = min{y | f(x_1, ..., x_n, y) = 0} 也是遞羅函數（如果這樣的y存在的話）。

使用原始遞歸定義的函數稱為「原始遞歸函數」。然而，原始遞歸函數並不能捕捉所有的可計算函數。例如，阿克曼函數（Ackermann function）是一個眾所周知的可計算函數，但它不是原始遞歸的。我們需要加入「最小化」運算來得到所有的遞歸函數。

克里尼的定理說：函數f是可由Lambda Calculus計算的，當且僅當f是遞歸函數。

這個定理的「⇐」方向（遞歸函數 → Lambda Calculus）可以通過具體構造來證明。我們可以為每個初始函數構造對應的λ項，然後說明如何處理複合、原始遞歸和最小化運算。

「⇒」方向（Lambda Calculus → 遞歸函數）的證明更為技術性。我們需要證明每個λ表達式的求值過程都可以被遞歸函數模擬。這可以通過哥德爾編碼的技術來實現：對每個λ表達式，我們可以計算它的哥德爾數，而求值過程可以通過遞歸函數來模擬。

這個結果意義深遠。它表明，我們有兩種看似完全不同的方式來定義「可計算性」——一種是直觀的遞歸函數概念，一種是抽象的Lambda Calculus——而它們是等價的。這進一步支持了丘奇-圖靈論題。

## 4.11 小結

本章我們詳細介紹了Lambda Calculus的數學理論。我們從計算的本質問題開始，討論了丘奇發明Lambda Calculus的動機。我們介紹了Lambda Calculus的語法，包括α轉換和β歸約兩種核心操作。我們還討論了不同的求值策略，介紹了Church-Rosser定理，並探討了停機問題的不可判定性。最後，我們說明了Lambda Calculus與遞歸函數的等價性。

Lambda Calculus雖然只是一個極簡的形式系統，卻具有極其深刻的理論意義。它不僅是計算理論的基石，也對程序設計語言的發展產生了深遠的影響。今天的函數式編程語言，如Lisp、Haskell、OCaml等，都可以直接追溯到Lambda Calculus。
