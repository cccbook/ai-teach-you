# 第十章：NP-Complete 與計算複雜度理論

## 10.1 計算複雜度理論的興起

計算複雜度理論是理論計算機科學的核心分支，它研究計算問題的「難度」——即解決問題所需的時間和空間資源。與可計算性理論（它告訴我們哪些問題是可解決的，哪些是不可解決的）不同，複雜度理論試圖量化「可解決」問題的難度。

計算複雜度理論的興起可以追溯到1960年代。1962年，哈特馬尼斯（Juris Hartmanis）和斯特恩斯（Richard Stearns）發表了開創性論文「論算法的計算複雜度」（On the Computational Complexity of Algorithms），這通常被認為是複雜度理論作為獨立數學領域的開端。他們引入了時間複雜度和空間複雜度的概念，開始系統地研究不同複雜度類之間的關係。

在複雜度理論發展的早期，一個重要的突破是1965年埃德蒙茲（Jack Edmonds）的工作。埃德蒙茲提出了「多項式時間」的概念，並認為「好」的算法應該在多項式時間內運行。他寫道：「從某種直觀的、審美的角度來看，多項式時間似乎就是『好』算法的代名詞。」這種觀點後來成為了複雜度理論的基本假設。

1960年代末和1970年代初，庫克（Stephen Cook）、萊文（Leonid Levin）和卡普（Richard Karp）的工作標誌著 NP-完全性理論的誕生。這些研究者證明了一大批重要的問題是「NP-完全」的，如果其中任何一個問題可以在多項式時間內解決，那麼所有 NP 問題都可以在多項式時間內解決。這個發現徹底改變了我們對計算困難性的理解。

## 10.2 時間複雜度與空間複雜度

在深入討論 NP-問題之前，我們需要建立一些基本概念。首先是時間複雜度和空間複雜度的定義。

**時間複雜度（Time Complexity）** 指的是算法執行所需的時間步驟數，以輸入規模的函數表示。給定一個算法 A 和一個輸入 x，令 T_A(x) 為算法 A 在輸入 x 上執行所需的步驟數。那麼 A 的時間複雜度是函數 T_A(n) = max{|x| = n} T_A(x)，即最壞情況下對所有長度為 n 的輸入所需的最大步驟數。

**空間複雜度（Space Complexity）** 指的是算法執行過程中所需的最大「內存」量，即算法使用的紙帶格子數或寄存器數。同樣地，空間複雜度也是輸入規模的函數。

在複雜度理論中，我們使用「漸進記號」來描述複雜度。最重要的記號包括：

- **大 O 記號** O(f(n))：表示上界，意思是存在常數 c 和 n₀，使得對所有 n ≥ n₀，複雜度 ≤ c·f(n)。
- **大 Ω 記號** Ω(f(n))：表示下界，意思是存在常數 c 和 n₀，使得對所有 n ≥ n₀，複雜度 ≥ c·f(n)。
- **大 Θ 記號** Θ(f(n))：表示同階，意思是既是上界又是下界。

例如，我們說排序算法具有 O(n log n) 的時間複雜度，意思是存在某個常數 c，使得對足夠大的 n，該算法最多需要 c·n log n 步。

## 10.3 P 類與 NP 類

P 類和 NP 類是計算複雜度理論中兩個最基本的複雜度類。

**P 類**（Polynomial Time）是所有可以在多項式時間內由確定性圖靈機解決的問題的集合。形式上，P = ∪_k TIME(n^k)，其中 TIME(t(n)) 是所有在時間 t(n) 內可判定的語言的集合。

P 類問題被認為是「易」問題的集合。如果一個問題屬於 P，我們就認為它有一個「好」的算法——算法運行時間不會隨輸入規模增長得太快。多項式時間通常被視為「可實用」的門檻：即使輸入規模很大，多項式時間算法仍然可能在合理時間內完成。

P 類包括許多常見的問題：
- 兩數相加（線性時間）
- 排序（O(n log n) 時間）
- 圖的可達性（圖的遍歷，O(n + m) 時間）
- 命題邏輯的可滿足性（SAT 的特殊情況）

**NP 類**（Nondeterministic Polynomial Time）是所有可以在多項式時間內由非確定性圖靈機解決的問題的集合。形式上，NP = ∪_k NTIME(n^k)，其中 NTIME(t(n)) 是所有在非確定性圖靈機的 O(t(n)) 時間內可判定的語言的集合。

NP 類問題的特點是：給定一個候選解答，我們可以在多項式時間內驗證這個解答是否正確。換句話說，「幸運地猜測」一個正確解答可以在多項式時間內驗證。

例如，考慮「哈密頓路徑」問題：給定一個圖，是否存在一條經過每個頂點恰好一次的路徑？如果有人給我們一條路徑，我們可以很容易地在多項式時間內驗證它是否是哈密頓路徑——我們只需要檢查路徑是否訪問了每個頂點恰好一次。然而，找到這樣一條路徑本身可能非常困難。

顯然，P ⊆ NP——任何可以在多項式時間內確定的問題當然也可以在多項式時間內驗證。然而，P 是否等於 NP 是計算機科學中最重要的未解決問題之一。

## 10.4 NP 完全性理論

NP-完全性理論是計算複雜度理論中最深刻的結果之一。它由庫克（Stephen Cook）和卡普（Richard Karp）在1970年代初獨立地發現。

**庫克-萊文定理（Cook-Levin Theorem）** 是 NP-完全性理論的基石。1971年，庫克證明了命題邏輯的可滿足性問題（SAT）是 NP-完全的。具體來說，他證明了：

- SAT 屬於 NP
- SAT 是 NP-困難的：任何 NP 問題都可以在多項式時間內歸約到 SAT

這意味著，如果我們有一個算法可以在多項式時間內解決 SAT，那麼我們就可以在多項式時間內解決任何 NP 問題。萊文獨立地獲得了類似的結果。

**NP-完全（NP-Complete）** 的正式定義如下：一個問題 L 是 NP-完全的，如果：
1. L 屬於 NP（可以在多項式時間內驗證解答）
2. 任何 NP 問題都可以在多項式時間內歸約到 L

「多項式時間歸約」是一種將一個問題轉化為另一個問題的方法。形式上，語言 A 可以多項式時間歸約到語言 B，如果存在一個多項式時間可計算的函數 f，使得 x ∈ A 當且僅當 f(x) ∈ B。

**卡普的二十一三個問題**。在庫克的論文發表後不久，1972年，卡普發表了著名的論文「繼承問題的複雜度」（Reducibility Among Combinatorial Problems），證明了21個經典的組合優化問題是 NP-完全的。這些問題包括：

- 頂點覆蓋問題（Vertex Cover）
- 旅行商問題（Traveling Salesman Problem）
- 哈密頓路徑問題（Hamiltonian Path）
- 集合覆蓋問題（Set Cover）
- 背包問題（Knapsack）
- 圖著色問題（Graph Coloring）

卡普的工作表明，NP-完全問題在現實世界中是普遍存在的。從那時起，已有數以千計的問題被證明是 NP-完全的。

## 10.5 P 與 NP：世紀難題

P vs NP 問題是計算機科學中最著名的未解決問題，也是克萊數學研究所提出的七個「千禧年大獎問題」之一。這個問題的表述非常簡單：P = NP 嗎？

表面上，這個問題只是問「易」問題和「難」問題是否相同。但實際上，它的影響遠遠超出計算機科學的範圍。如果 P = NP 被證明為真，它將會徹底改變我們的世界：

- 密碼學將不復存在：所有的加密系統都將被破解，因為因式分解等問題可以在多項式時間內解決。
- 數學證明可以自動化：找到數學定理的證明和驗證證明是一樣困難的，如果 P = NP，我們就可以在多項式時間內找到證明。
- 人工智能將取得突破：許多現在認為困難的 AI 問題（如自然語言理解、模式識別）將變得 tractable。

另一方面，如果 P ≠ NP 被證明，它將確認我們的直觀：確實存在本質上困難的問題，我們需要近似算法或启发式方法來處理它們。

**為什麼 P vs NP 很難？** 這個問題的困難在於，我們很難證明一個問題不能以某種更快的方式解決。要證明 P ≠ NP，需要排除所有可能的多項式時間算法——這幾乎是不可能的。同時，我們也沒有找到任何可以在多項式時間內解決 NP-完全問題的算法。

大多數計算機科學家相信 P ≠ NP。這個信念的一個主要理由是：如果 P = NP是真的，我們早就會發現它。數十年來，數學家和計算機科學家一直在尋找更快的方法來解決 NP-完全問題，但沒有成功。

## 10.6 著名的 NP 完全問題

讓我們更詳細地了解一些最著名的 NP 完全問題。

**命題邏輯的可滿足性問題（SAT）**：這是第一個被證明是 NP-完全的問題。SAT 問的是：給定一個命題邏輯公式，是否存在一種對變量的真值賦值使得整個公式為真？例如，公式 (p ∨ q) ∧ (¬p ∨ r) ∧ (¬q ∨ ¬r) 是可滿足的（p = true, q = false, r = true 可以滿足它）。

**3-SAT**：這是 SAT 的一個重要特殊情況，其中每個子句恰好包含三個文字。3-SAT 仍然是 NP-完全的，這一事實使它成為證明其他問題是 NP-完全的便利工具。

**頂點覆蓋問題（Vertex Cover）**：給定一個圖 G 和整數 k，是否存在一個大小不超過 k 的頂點集合，使得圖中的每一條邊至少與集合中的一個頂點相鄰？這個問題在網絡設計、異常檢測等領域有應用。

**哈密頓路徑問題**：給定一個圖，是否存在一條經過每個頂點恰好一次的路徑？這個問題與「旅行商問題」密切相關，後者要求找到一條最短的哈密頓路徑。

**旅行商問題（TSP）**：這是組合優化中最著名的問題之一。給定一組城市和它們之間的距離，找到經過所有城市一次並返回起點的最短路徑。TSP 的決策版本（是否存在長度不超過 L 的路徑）是 NP-完全的。

**背包問題（Knapsack）**：給定一組物品的重量和價值，以及一個容量限制，是否可以選擇物品使得總重量不超過容量且總價值至少為指定值？

**圖著色問題**：給定一個圖 G 和整數 k，是否可以用 k 種顏色給頂點著色，使得相鄰的頂點有不同的顏色？

## 10.7 處理 NP 困難問題的策略

既然 NP-完全問題不可能在多項式時間內完全解決，我們需要其他策略來處理它們。

**近似算法（Approximation Algorithm）**：近似算法在多項式時間內找到一個「接近最優」的解決方案。例如，對於背包問題，我們可以設計一個近似算法，使得解決方案的價值至少是最優價值的 90%（對於任何輸入）。

**启发式算法（Heuristic Algorithm）**：启发式算法是基於直觀或經驗的算法，不能保證最優性，但在實踐中往往表現良好。例如，模擬退火、遺傳算法、蟻群優化等元启发式算法被廣泛應用於解決組合優化問題。

**參數化算法（Parameterized Algorithm）**：參數化算法接受額外的參數 k，並在 f(k)·poly(n) 時間內運行，其中 f 是某個函數，poly 是多項式。例如，頂點覆蓋問題有一個經典的參數化算法，運行時間為 O(1.2738^k + kn)，其中 k 是覆蓋的大小。

**指數時間算法**：對於小型輸入，我們可以使用明確的指數時間算法。例如，使用動態規劃的 O(n·2^n) 算法可以解決 TSP 的精確版本。

**量子計算**：量子計算機有潛力在某些 NP 問題上提供加速。Grover 算法可以在 O(√N) 時間內搜索無序數據庫，這比經典的 O(N) 有二次加速。然而，量子計算機不太可能將 NP-完全問題帶入 P。

## 10.8 複雜度類的其他重要概念

除了 P 和 NP，複雜度理論中還有許多其他重要的複雜度類。

**NP-困難（NP-Hard）**：NP-困難問題是「至少和 NP-完全問題一樣難」的問題。形式上，一個問題 L 是 NP-困難的，如果任何 NP 問題都可以在多項式時間內歸約到 L。NP-困難問題不一定屬於 NP——它們可能更難。

**co-NP**：co-NP 是 NP 的「補類」。一個語言 L 屬於 co-NP，如果它的補語言 ¬L 屬於 NP。換句話說，co-NP 中的問題是「很容易驗證無解」的問題。顯然，P ⊆ NP ∩ co-NP，但我們不知道 NP 是否等於 co-NP。

**PSPACE**：PSPACE 是所有可以在多項式空間內由確定性圖靈機解決的問題。顯然，NP ⊆ PSPACE，但 PSPACE 是否等於 NP 是另一個未解決的問題。交替式圖靈機和博奕論與 PSPACE 密切相關。

**EXPTIME**：EXPTIME 是所有可以在指數時間內解決的問題的集合。顯然，PSPAPE ⊆ EXPTIME。我們知道 P ≠ EXPTIME（由時間層次定理），但 P 是否等於 PSPACE 仍未解決。

**BPP**：BPP 是所有可以在多項式時間內以高概率解決的問題，允許雙向誤差（bounded error）。BPP 被認為是「可概率有效計算」的類。，我們不知道 BPP 是否等於 P，但大多數研究者認為它們是相等的。

## 10.9 小結

本章我們介紹了計算複雜度理論，特別是 NP-完全性理論。我們定義了時間複雜度和空間複雜度的概念，介紹了 P 類和 NP 類的基本定義，詳細討論了 NP-完全性理論的誕生、庫克-萊文定理和卡普的貢献。我們還探討了 P vs NP 這個世紀難題，並列舉了一些著名的 NP-完全問題，最後討論了處理 NP-困難問題的各種策略。

NP-完全性理論是計算機科學中最深刻的理論之一。它告訴我們，大量的實際問題在計算上是「困難」的，我們不太可能找到解決它們的多項式時間算法。這個結論對算法設計、密碼學、人工智能等領域都有深遠的影響。

在接下來的章節中，我們將繼續探索計算理論的更多前沿領域，包括函數式編程與 Lambda 的遺產、類型理論、領域特定語言、量子計算理論等。這些主題將展示計算理論如何影響現代計算機科學的發展。
