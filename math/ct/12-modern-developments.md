## 12.1 量子計算

### 12.1.1 量子計算的興起

1982年，物理學家理查德·費曼（Richard Feynman）提出：
> 「量子系統的模擬對經典計算機來說是困難的，但如果使用量子計算機呢？」

這個問題開啟了量子計算的新時代。

### 12.1.2 量子計算的基礎

量子計算基於量子力學的幾個關鍵原理：

```
┌────────────────────────────────────────────────────────────────┐
│                    量子計算基礎                                │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  量子位 (Qubit)                                               │
│  ├── 0 和 1 的疊加態                                         │
│  └── |ψ⟩ = α|0⟩ + β|1⟩，其中 |α|² + |β|² = 1              │
│                                                                │
│  量子疊加 (Superposition)                                     │
│  ├── 多個狀態同時存在                                         │
│  └── n個量子位 = 2ⁿ 個狀態的疊加                            │
│                                                                │
│  量子糾纏 (Entanglement)                                      │
│  ├── 粒子之間的強關聯                                         │
│  └── 測量一個粒子立即影響另一個                                │
│                                                                │
│  量子干預 (Interference)                                      │
│  ├── 利用振幅放大正確答案                                     │
│  └── 消滅錯誤答案                                             │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 12.1.3 量子計算的複雜度

量子計算為某些問題提供了指數級加速：

| 問題 | 經典複雜度 | 量子複雜度 |
|------|-----------|-----------|
| 整數分解 | $O(e^{n^{1/3}})$ | $O(n^3)$ (Shor) |
| 資料庫搜索 | $O(N)$ | $O(\sqrt{N})$ (Grover) |
| 模擬量子系統 | $O(2^n)$ | $O(n^k)$ |
| SAT | $O(2^n)$ | $O(2^{n/2})$ |

### 12.1.4 量子計算的意義

量子計算並不意味著P = NP。它仍然受圖靈機的限制，只是提供了不同的計算方式。

```python
"""
量子計算基礎模擬
注意：這只是概念演示，真正的量子計算需要量子硬體
"""

from typing import Tuple, List
import math
import cmath

class Qubit:
    """量子位（概念模擬）"""
    
    def __init__(self, alpha: complex = 1, beta: complex = 0):
        # 歸一化
        norm = math.sqrt(abs(alpha)**2 + abs(beta)**2)
        self.alpha = alpha / norm
        self.beta = beta / norm
    
    def measure(self) -> int:
        """測量量子位，返回0或1"""
        prob_zero = abs(self.alpha)**2
        return 0 if math.random() < prob_zero else 1
    
    def __repr__(self):
        return f"Qubit(α={self.alpha:.3f}, β={self.beta:.3f})"


class QuantumGate:
    """量子門（概念模擬）"""
    
    @staticmethod
    def hadamard(qubit: Qubit) -> Qubit:
        """Hadamard門：創建疊加態"""
        alpha_new = (qubit.alpha + qubit.beta) / math.sqrt(2)
        beta_new = (qubit.alpha - qubit.beta) / math.sqrt(2)
        return Qubit(alpha_new, beta_new)
    
    @staticmethod
    def pauli_x(qubit: Qubit) -> Qubit:
        """Pauli-X門：量子NOT"""
        return Qubit(qubit.beta, qubit.alpha)
    
    @staticmethod
    def cnot(control: Qubit, target: Qubit) -> Tuple[Qubit, Qubit]:
        """CNOT門：控制-NOT"""
        if control.measure() == 1:
            return control, Qubit(target.beta, target.alpha)
        return control, target


def demonstrate_quantum_principles():
    """演示量子計算原理"""
    print("=== 量子計算基礎演示 ===\n")
    
    print("1. 創建初始量子位 |0⟩：")
    q0 = Qubit(1, 0)
    print(f"   {q0}")
    print(f"   測量結果：{'0' if q0.measure() == 0 else '1'} (100% 為0)")
    
    print("\n2. 應用Hadamard門，創建疊加態：")
    q = Qubit(1, 0)
    q_h = QuantumGate.hadamard(q)
    print(f"   {q_h}")
    print(f"   測量：{'0' if q_h.measure() == 0 else '1'} (50% 概率)")
    
    print("\n3. Grover搜索算法概念：")
    print("   經典搜索：O(N)")
    print("   Grover搜索：O(√N)")
    print("   原理：")
    print("   - 將目標狀態的振幅放大")
    print("   - 減小非目標狀態的振幅")
    print("   - 重複應用 oracle + 擴散")
    
    print("\n4. Shor算法概念：")
    print("   用於整數分解")
    print("   經典：亞指數時間")
    print("   量子：多項式時間")
    print("   這對RSA加密有重大影響")


demonstrate_quantum_principles()
```

## 12.2 複雜度理論的新發展

### 12.2.1 PCP定理

1990年代，PCP（Probabilistically Checkable Proof）定理的證明是理論電腦科學的重大突破：

> **PCP定理**
>
> 所有NP問題都有概率可驗證的證明，且驗證者只讀取證明的固定位數。

這個定理與近似算法的難度密切相關：

```
┌────────────────────────────────────────────────────────────────┐
│                    PCP定理的意義                               │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  如果一個優化問題很難近似到某個因素，                          │
│  那麼這個問題可能是NP-困難的                                  │
│                                                                │
│  例如：MAX-3SAT                                                │
│  - 簡單貪心：3/4 的最優解                                      │
│  - PCP定理暗示：無法用多項式時間得到更好的近似                 │
│    （除非P = NP）                                             │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 12.2.2 電路複雜度

電路複雜度研究用電路（邏輯閘網絡）計算函數所需的資源：

- **電路大小**：使用的閘數量
- **電路深度**：從輸入到輸出的最長路徑
- **閘的類型**：AND、OR、NOT等

電路複雜度與時間複雜度有深刻聯繫：

$$P \subseteq SIZE(poly(n)) \subseteq P/poly$$

### 12.2.3 證明複雜度

證明複雜度研究各種證明系統的強度：

| 系統 | 描述 | 與NP的關係 |
|------|------|-----------|
| 命題邏輯 | 布爾公式 | 與NP關係不明確 |
| 零知識證明 | 不洩露知識的證明 | 密碼學應用 |
| 機率證明 | PCP | NP = PCP(log n, O(1)) |
| 代數證明 | 多項式恆等 | 格密碼學 |

## 12.3 新的計算範式

### 12.3.1 DNA計算

1994年，萊恩·阿德曼（Leonard Adleman）用DNA分子解決了哈密頓路徑問題。

DNA計算的優點：
- 高度並行（數十億分子同時計算）
- 低能耗
- 可自我複製

缺點：
- 規模限制
- 錯誤率高
- 難以控制

### 12.3.2 模糊計算

模糊計算處理不精確、不確定性問題：

```
┌────────────────────────────────────────────────────────────────┐
│                    模糊邏輯 vs 精確邏輯                        │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  精確邏輯：              模糊邏輯：                            │
│  ─────────              ─────────                              │
│  溫度 > 30°C = 熱      溫度30°C = 0.7熱                     │
│  溫度 ≤ 30°C = 不熱    溫度25°C = 0.3熱                     │
│                                                                │
│  離散、是/否           連續、多少                             │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 12.3.3 進化計算

進化計算模擬自然選擇的過程：

1. **遺傳演算法**：選擇、交叉、突變
2. **演化策略**：自適應參數
3. **遺傳編程**：演化程式結構

### 12.3.4 儲水池計算

儲水池計算利用動力系統的暫態行為進行計算：

- 輸入映射到動力系統的暫態
- 讀取系統狀態作為輸出
- 適用於時序信號處理

## 12.4 計算理論與人工智慧

### 12.4.1 圖靈測試

1950年，圖靈提出了著名的「圖靈測試」：

> 如果一台機器能夠與人類對話而不被識別為機器，那麼這台機器可以被認為是「智能」的。

圖靈測試的爭議：
- 它測試的是欺騙能力，不是智能
- 它忽視了智能的其他方面
- 它暗示語言是智能的唯一標準

### 12.4.2 機器學習的複雜度

機器學習提出了新的計算問題：

| 問題 | 複雜度 |
|------|--------|
| 學習一個有限的假設 | 多項式時間 |
| PAC學習一般概念 | 可能需要指數時間 |
| 神經網絡訓練 | NP-困難 |
| 加強學習 | PSPACE-困難 |

### 12.4.3 深度學習的理論解釋

深度學習的成功至今缺乏完整的理論解釋：

- **表示能力**：深網絡可以表示複雜函數
- **學習能力**：梯度下降為什麼有效？
- **泛化能力**：為什麼訓練資料有限但泛化好？

## 12.5 可計算性理論的新進展

### 12.5.1 維納爾斯潘定理

维纳尔斯潘（Wienerspan）定理研究可判定性問題的邊界：

> 存在絕對不可判定的問題
> —— 即使用所有可能的圖靈機 oracle 也無法解決

### 12.5.2 隨機性與計算

演算法隨機性研究什麼是「隨機」序列：

- **統計隨機性**：通過所有統計測試
- **複雜性隨機性**：柯氏複雜性高
- **不可壓縮性**：無法用比自身更短的程式生成

### 12.5.3 逆數學

逆數學研究證明數學定理所需的公理系統：

> 什麼樣的公理系統才能證明某個數學定理？

這與哥德爾定理有深刻聯繫。

## 12.6 密碼學與計算理論

### 12.6.1 現代密碼學的基礎

現代密碼學基於計算假設：

| 假設 | 安全性基礎 |
|------|-----------|
| 大數分解困難 | RSA加密 |
| 離散對數困難 | DH密鑰交換、橢圓曲線 |
| 格的最近向量問題 | 後量子密碼 |
| 單向函數存在 | 理論基礎 |

### 12.6.2 同態加密

同態加密允許在密文上直接計算：

```
┌────────────────────────────────────────────────────────────────┐
│                    同態加密的層級                               │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  部分同態：只支持一種運算                                      │
│  └─ 加法或乘法                                                │
│                                                                │
│  層級同態：支持有限次的加法和乘法                              │
│  └─ BGN方案、格拉斯曼加密                                     │
│                                                                │
│  全同態：支持任意次數的加法和乘法                              │
│  └─ Gentry的突破性工作（2009）                                │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 12.6.3 後量子密碼學

量子電腦威脅現有加密系統，後量子密碼學研究可以抵抗量子攻擊的方案：

- 基於格的密碼
- 基於編碼的密碼
- 基於哈希的簽名
- 多變量多項式密碼
