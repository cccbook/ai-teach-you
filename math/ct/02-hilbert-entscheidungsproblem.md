# 第 2 章：希爾伯特與判定問題

## 概述

1900年，在巴黎舉行的第二屆國際數學家大會上，一位德國數學家站在講台上，發表了一篇改變數學歷史的演講。他提出了23個未解決的數學問題，這些問題將指引整個二十世紀的數學發展方向。

這位數學家就是**大衛·希爾伯特**（David Hilbert），而他的第10個問題，為後來的計算理論埋下了種子。

## 2.1 大衛·希爾伯特：數學世界的建築師

### 2.1.1 早年與崛起

1862年，希爾伯特出生於普魯士的哥尼斯堡（現在的俄羅斯加里寧格勒），這裡也是哲學家康德的故鄉。希爾伯特從小就展現出驚人的數學天賦，但他的才華並非那種「神童」式的爆發，而是一種持久、深刻、系統性的智慧。

1885年，23歲的希爾伯特獲得博士學位。1895年，他成為哥廷根大學教授，這個職位在當時是世界數學的中心。接下來的三十年，希爾伯特將哥廷頓打造成為無可爭議的全球數學首都。

### 2.1.2 希爾伯特的三大信念

希爾伯特對數學有著近乎宗教般的信念，可以用三個關鍵詞概括：

```
┌────────────────────────────────────────────────────────────────┐
│                    希爾伯特的數學哲學                            │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  1. 形式主義 (Formalism)                                       │
│     ─────────────────                                          │
│     數學是符號操作的系統                                       │
│     真理存在於符號之間的邏輯關係                                │
│                                                                │
│  2. 完美主義 (Perfectionism)                                   │
│     ─────────────────                                          │
│     每個數學問題都有答案                                       │
│     要麼被證明，要麼被否證                                     │
│     「我們必須知道，我們將會知道」                              │
│                                                                │
│  3. 系統化 (Systematization)                                    │
│     ─────────────────                                          │
│     數學應該是一個封閉、完整的形式系統                         │
│     所有真理都可以從少數公理推導出來                            │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

這三個信念將在1931年被庫爾特·哥德爾徹底動搖，但這是後話了。讓我們先把時間撥回1900年。

## 2.2 1900年：23個問題

### 2.2.1 巴黎演說

1900年8月8日，希爾伯特在巴黎國際數學家大會上發表演說。根據他自己的說法，他選擇問題的標準是：

> 「在科學史上，解決一個重要問題會帶來新的真理領域的發現。」

這23個問題影響深遠，至今仍有半數以上未完全解決。讓我們看看其中最重要的幾個：

| 問題編號 | 問題內容 | 現狀 |
|----------|----------|------|
| 第1題 | 康托爾的連續統假設 | 部分解決（1963年） |
| 第2題 | 算術公理的一致性 | 哥德爾（1931）、根岑（1936）證明不可完全證明 |
| 第10題 | 丢番圖方程的可判定性 | 馬蒂塞維奇（1970）否定解決 |
| 第23題 | 數學物理的公理化 | 部分完成 |

### 2.2.2 希爾伯特的第10問題

第10問題的原文是：

> **第10問題（Entscheidungsproblem / 判定問題）**
>
> 給定一個任意多變數的整係數代數方程，是否存在一個有限步驟的演算法，來判斷這個方程是否有整數解？

這個問題看起來很技術性，但它蘊含著一個深刻的問題：

> **是否存在一個「萬能機械」，能夠判定數學命題的真假？**

這就是後來著名的**判定問題**（Entscheidungsproblem）。

## 2.3 判定問題的由來

### 2.3.1 從萊布尼茲到希爾伯特

「判定問題」的想法並非希爾伯特原創，它源於十七世紀的哲學家兼數學家**戈特弗里德·萊布尼茲**（Gottfried Leibniz）。

萊布尼茲設想過一種「通用語言」（characteristica universalis），可以表達所有數學和邏輯概念。他甚至設想過一種「推理機器」（calculus ratiocinator），能夠自動推導數學定理。

這個夢想一直活在數學家的心中，直到希爾伯特在1928年正式提出：

> **判定問題**：是否存在一個演算法，對於任意一階邏輯命題，都能判定其是否普遍有效？

### 2.3.2 形式系統的基本要素

希爾伯特的「判定問題」基於一個重要的概念：**形式系統**。

一個形式系統由以下要素組成：

```
┌─────────────────────────────────────────────────────────────────┐
│                       形式系統的構成                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 符號表 (Alphabet)                                          │
│     用於構造公式的字符集合                                       │
│     例如：¬, ∧, ∨, →, ∀, ∃, x, y, z, (, )                      │
│                                                                 │
│  2. 語法規則 (Syntax)                                           │
│     定義什麼是「合法的公式」                                     │
│     例如：∀x(P(x) → Q(x)) 是合法的                              │
│                                                                 │
│  3. 公理 (Axioms)                                               │
│     被接受為真的基本命題                                         │
│     例如：(A → (B → A))                                        │
│                                                                 │
│  4. 推理規則 (Inference Rules)                                  │
│     從已知命題推導新命題的規則                                   │
│     例如：肯定前件 (Modus Ponens)                               │
│           如果 A 和 A→B 為真，則 B 為真                          │
│                                                                 │
│  5. 證明 (Proof)                                                │
│     從公理出發，通過推理規則導出的命題序列                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3.3 一階邏輯

一階邏輯（First-Order Logic）是希爾伯特形式系統的核心。它比命題邏輯更強大，因為它允許**量化**：

- **全稱量化** (∀)：對所有...
- **存在量化** (∃)：存在...

例如，下面是一個典型的一階邏輯命題：

$$\forall x \forall y \forall z [(x + y) + z = x + (y + z)]$$

翻譯成自然語言：**對於所有 x, y, z，加法運算都是結合的。**

## 2.4 判定問題的精確定義

### 2.4.1 什麼是「判定」？

判定問題要求的是一個**決策程序**（decision procedure），這是一個滿足以下條件的演算法：

1. **明確性**：每一步操作都是明確規定的
2. **有限性**：程序必須在有限步驟內終止
3. **正確性**：給定正確的輸入，總是輸出正確的答案

### 2.4.2 判定問題的答案

判定問題的答案是什麼？讓我們先預告結論：

## 2.5 希爾伯特計劃的崩潰

### 2.5.1 三個階段

希爾伯特有一個宏偉的「形式化計劃」：

```
┌────────────────────────────────────────────────────────────────┐
│                    希爾伯特的三大目標                            │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  1. 一致性 (Consistency)                                      │
│     證明數學不會產生矛盾                                        │
│     形式系統內不會同時證明 P 和 ¬P                              │
│                                                                │
│  2. 完整性 (Completeness)                                      │
│     證明所有數學真理都可以被證明                                │
│     每個真命題都是系統內的可證命題                               │
│                                                                │
│  3. 可判定性 (Decidability)                                    │
│     存在一個演算法，能判定任意命題的真假                         │
│     數學真理可以被「計算」出來                                  │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 2.5.2 哥德爾的第一擊：不完備

1931年，25歲的庫爾特·哥德爾發表了不完備定理，給了希爾伯特計劃第一記重拳：

> **哥德爾第一不完備定理**
>
> 任何足夠強大的形式系統，都存在無法在系統內被證明或否證的命題。

這直接否定了希爾伯特的「完整性」目標。

### 2.5.3 哥德爾的第二擊：一致性

更糟糕的是，哥德爾還證明了：

> **哥德爾第二不完備定理**
>
> 任何足夠強大的形式系統，無法在系統內部證明自身的一致性。

這意味著，我們無法用數學方法向自己證明「數學不會產生矛盾」。

### 2.5.4 圖靈的最後一擊：不可判定

1936年，圖靈的論文進一步證明：

> 判定問題的答案是否定的。不存在能夠判定所有數學命題的通用演算法。

希爾伯特的三大目標，全部落空。

## 2.6 一個具體的例子：丟番圖方程

讓我們用具體例子來理解第10問題——丢番圖方程的可判定性。

[程式檔案：02-1-diophantine.py](../_code/02/02-1-diophantine.py)

```python
"""
丢番圖方程（Diophantine Equation）
形如：a*x + b*y = c 的整數方程

第10問題問的是：是否存在通用演算法，
判斷任意丢番圖方程是否有整數解？
"""

def gcd(a: int, b: int) -> int:
    """計算最大公約數"""
    while b:
        a, b = b, a % b
    return abs(a)

def extended_gcd(a: int, b: int) -> tuple:
    """擴展歐幾里得算法：返回 (gcd, x, y) 使得 ax + by = gcd"""
    if b == 0:
        return (abs(a), 1 if a > 0 else -1, 0)
    else:
        gcd, x1, y1 = extended_gcd(b, a % b)
        x = y1
        y = x1 - (a // b) * y1
        return (gcd, x, y)

def has_integer_solution(a: int, b: int, c: int) -> tuple:
    """
    判斷方程 ax + by = c 是否有整數解
    返回：(是否有解, 其中一組解或None)
    
    定理：ax + by = c 有整數解 當且僅當 gcd(a,b) 整除 c
    """
    if a == 0 and b == 0:
        return (c == 0, None)
    
    g = gcd(a, b)
    
    if c % g != 0:
        return (False, None)
    
    # 找到一組解
    a_reduced, b_reduced = a // g, b // g
    c_reduced = c // g
    
    _, x0, y0 = extended_gcd(a_reduced, b_reduced)
    
    x0 *= c_reduced
    y0 *= c_reduced
    
    return (True, (x0, y0))

def general_solution(a: int, b: int, c: int) -> str:
    """給出通解公式"""
    has_sol, sol = has_integer_solution(a, b, c)
    
    if not has_sol:
        return f"方程 {a}x + {b}y = {c} 沒有整數解"
    
    x0, y0 = sol
    g = gcd(a, b)
    
    if b == 0:
        return f"x = {x0}"
    elif a == 0:
        return f"y = {y0}"
    else:
        return (f"方程 {a}x + {b}y = {c} 的通解為：\n"
                f"  x = {x0} + {b//g}t\n"
                f"  y = {y0} - {a//g}t\n"
                f"  其中 t 是任意整數")


# 測試案例
test_cases = [
    (2, 3, 7),    # 2x + 3y = 7
    (4, 2, 6),    # 4x + 2y = 6
    (6, 4, 5),    # 6x + 4y = 5
    (0, 5, 15),   # 0x + 5y = 15
    (3, 0, 9),    # 3x + 0y = 9
    (0, 0, 0),    # 0x + 0y = 0
    (0, 0, 5),    # 0x + 0y = 5
]

for a, b, c in test_cases:
    print(general_solution(a, b, c))
    print()
```

執行結果：

```
方程 2x + 3y = 7 的通解為：
  x = -14 + 3t
  y = 10 - 2t
  其中 t 是任意整數

方程 4x + 2y = 6 的通解為：
  x = 3 + t
  y = -3 - 2t
  其中 t 是任意整數

方程 6x + 4y = 5 沒有整數解

方程 0x + 5y = 15 的通解為：
y = 3

方程 3x + 0y = 9 的通解為：
x = 3

方程 0x + 0y = 0 的通解為：
任意 x, y

方程 0x + 0y = 5 沒有整數解
```

### 2.6.1 為什麼簡單的方程可判定？

上面的例子展示了一個關鍵洞察：

> **簡單的線性丢番圖方程 ax + by = c 是可判定的**

因為我們有明確的數學理論（擴展歐幾里得算法）可以解決它。

### 2.6.2 為什麼一般的方程不可判定？

但一旦方程變得複雜，例如：

$$x_1^2 + x_2^2 + x_3^2 + \cdots + x_n^2 = k$$

這個問題最終被證明是**不可判定的**——1970年，蘇聯數學家**尤里·馬蒂塞維奇**（Yuri Matiyasevich）證明了這一點。
