# 第 6 章：圖靈機理論

## 概述

1936年，在英國劍橋的一間小公寓裡，一位年輕的數學家正在思考一個看似簡單的問題：

> **當人類數學家進行計算時，他的大腦在做什麼？**

這位數學家就是**艾倫·圖靈**（Alan Turing），他的答案徹底改變了我們對計算的認知。

圖靈的答案是：**計算機器的抽象模型**。這個模型後來被稱為**圖靈機**（Turing Machine），成為現代電腦科學的理論基石。

## 6.1 艾倫·圖靈：密碼破譯者與理論家

### 6.1.1 早年的天才

1912年6月23日，艾倫·圖靈出生於英國倫敦。從小學開始，圖靈就展現出驚人的數學天賦，但他同時也是一個害羞、內向的孩子。

1928年，16歲的圖靈遇到了同學克里斯托弗·莫坎（Christopher Morcom），一個同樣熱愛數學的少年。他們一起討論愛因斯坦的相對論，一起閱讀最新的科學論文。不幸的是，莫坎在1930年因肺結核去世，這對圖靈打擊巨大。

有人說，圖靈後來對機器智能的執著研究，部分源自於他想「讓機器思考」——也許是為了與逝去的摯友進行某種形式的對話。

### 6.1.2 劍橋與普林斯頓

1931年，圖靈進入劍橋大學國王學院，師從數學家戈弗雷·哈羅德·哈代（Godfrey Harold Hardy）。1936年，圖靈發表了他的成名之作《論可計算數及其在判定問題上的應用》。

這篇論文包含三個重要貢獻：
1. **圖靈機**的定義
2. **通用圖靈機**的概念
3. **停機問題**不可判定的證明

1936-1938年，圖靈來到普林斯頓大學訪問，師從丘奇。在這裡，他遇見了哥德爾，與這些偉大數學家進行了深入交流。

### 6.1.3 戰時工作：破譯恩尼格瑪

二戰期間，圖靈加入了布萊切利公園（Bletchley Park）的密碼破譯團隊。這裡聚集了英國最優秀的數學家、語言學家和填字遊戲高手，他們的任務是破譯納粹德軍的恩尼格瑪密碼機。

圖靈設計了「炸彈」（Bombe）——一種機電裝置，可以快速枚舉恩尼格瑪的可能設置。這項工作被認為將二戰縮短了至少兩年，拯救了無數生命。

戰後，圖靈繼續在曼徹斯特大學工作，設計早期電子電腦。但他的生命在1954年悲劇性地結束了——圖靈因同性戀傾向被定罪，兩年後死於氰化物中毒，年僅41歲。

## 6.2 圖靈機的直覺

### 6.2.1 圖靈的思考實驗

圖靈在設計圖靈機時，問了自己一個問題：

> **什麼是「可計算的」？**

他的方法是：如果一個「機器」能做的事，我們人類數學家在紙上也能做，那麼這就是一個好的計算模型。

圖靈觀察到，人類數學家進行計算時：
1. **在紙上寫下符號**（或擦除）
2. **移動注意力**到紙的另一部分
3. **記住當前的「心智狀態」**

圖靈將這些元素抽象化，發明了圖靈機。

### 6.2.2 圖靈機的組成

```
┌─────────────────────────────────────────────────────────────────┐
│                        圖靈機示意圖                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│                      ┌─────────────┐                            │
│                      │   控制單元   │                            │
│                      │  (有限狀態)  │                            │
│                      └──────┬──────┘                            │
│                             │                                   │
│                    讀取/寫入/移動                                │
│                             │                                   │
│    ◀──┬──▶  ┌──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──▶  ◀──┐ │
│        │    │  │▒▒│▒▒│  │▒▒│  │  │▒▒│  │  │  │▒▒│  │      │ │
│        │    └──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┘      │ │
│        │         紙帶（無限長）                                 │ │
│        │         ▲ 讀寫頭                                      │ │
│        │                                                          │
│        └──────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

圖靈機由四部分組成：

| 組成部分 | 描述 |
|---------|------|
| **紙帶** | 無限長的格子，每格可寫入一個符號 |
| **讀寫頭** | 可以在紙帶上讀取或寫入符號，可以左右移動 |
| **控制單元** | 有限狀態自動機，根據當前狀態和讀取的符號決定動作 |
| **狀態寄存器** | 記錄當前狀態 |

### 6.2.3 圖靈機的運作

圖靈機的運作非常簡單，重複執行以下循環：

```
┌─────────────────────────────────────────────────────────────────┐
│                      圖靈機運作循環                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 讀取：讀寫頭讀取當前格子中的符號                              │
│                                                                 │
│  2. 查找：根據 (當前狀態, 讀取的符號) 查表                       │
│                                                                 │
│  3. 寫入：可選地寫入一個新符號到當前格子                          │
│                                                                 │
│  4. 移動：可選地將讀寫頭向左或向右移動一格                        │
│                                                                 │
│  5. 轉換：進入一個新的狀態                                        │
│                                                                 │
│  如果進入特殊狀態「停機」(halt)，機器停止運作                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 6.3 圖靈機的形式定義

### 6.3.1 七元組定義

一台圖靈機可以用七元組表示：

$$M = (Q, \Sigma, \Gamma, \delta, q_0, q_{accept}, q_{reject})$$

其中：

| 符號 | 含義 | 描述 |
|------|------|------|
| $Q$ | 狀態集合 | 機器的所有可能狀態 |
| $\Sigma$ | 輸入字母表 | 不包含空格的輸入符號集合 |
| $\Gamma$ | 紙帶字母表 | 可寫入的所有符號集合，包含 $\Sigma$ 和空格 |
| $\delta$ | 轉換函數 | $Q \times \Gamma \rightarrow Q \times \Gamma \times \{L, R\}$ |
| $q_0$ | 起始狀態 | 機器的初始狀態 |
| $q_{accept}$ | 接受狀態 | 機器接受輸入的狀態 |
| $q_{reject}$ | 拒絕狀態 | 機器拒絕輸入的狀態 |

### 6.3.2 轉換函數

轉換函數 $\delta$ 是圖靈機的核心。它定義了在每種情況下機器的行為：

$$\delta(q, a) = (q', b, D)$$

表示：
- 當前狀態是 $q$
- 讀取的符號是 $a$
- 轉換為狀態 $q'$
- 寫入符號 $b$
- 讀寫頭向 $D$ 方向移動（$L$ = 左，$R$ = 右）

### 6.3.3 圖靈機的擴展

基本圖靈機只有一條紙帶。研究者定義了幾種擴展：

```
┌────────────────────────────────────────────────────────────────┐
│                      圖靈機的擴展                               │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  多帶圖靈機                                                     │
│  ├── 多條平行紙帶                                               │
│  └── 等價於單帶圖靈機（可模擬）                                │
│                                                                │
│  非確定性圖靈機 (NTM)                                           │
│  ├── 每個狀態-符號組合可以有多個轉換                            │
│  └── 等價於確定性圖靈機（可模擬）                              │
│                                                                │
│  枚舉器                                                         │
│  ├── 輸出紙帶用於枚舉語言                                       │
│  └── 等價於圖靈機                                              │
│                                                                │
│  所有這些擴展都與基本圖靈機計算等價                             │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

## 6.4 設計圖靈機

### 6.4.1 例子：識別回文

讓我們設計一個圖靈機，識別二進制回文（例如 "101" 和 "11" 是回文，"1010" 不是）。

**策略**：
1. 從左端開始
2. 讀取最左邊的符號，記住它
3. 擦除這個符號，移動到最右邊
4. 比較是否匹配
5. 重複直到相遇

**狀態設計**：
- $q_0$：起始，讀取左端第一個符號
- $q_1$：穿過中間，向右移動找右端
- $q_2$：比較符號
- $q_{accept}$：是回文
- $q_{reject}$：不是回文

### 6.4.2 例子：複製字串

讓我們設計一個圖靈機，將二進制字串複製一份（例如 "101" → "101101"）。

**策略**：
1. 從左端開始
2. 讀取並記住每個符號
3. 將讀到的符號寫到紙帶末尾
4. 繼續直到整個字串複製完成

[程式檔案：06-1-palindrome.py](../_code/06/06-1-palindrome.py)

```python
"""
圖靈機設計示例：識別回文
展示如何用狀態轉換圖描述圖靈機
"""

from typing import Dict, Tuple, Optional, List
from enum import Enum

class Direction(Enum):
    """讀寫頭移動方向"""
    LEFT = 'L'
    RIGHT = 'R'
    STAY = 'S'  # 原地不動

class TuringMachine:
    """圖靈機模擬器基類"""
    
    def __init__(self):
        self.tape: List[str] = []
        self.head: int = 0
        self.state: str = ""
        self.states: set = set()
        self.alphabet: set = set()
        self.tape_alphabet: set = set()
        self.transitions: Dict[Tuple[str, str], Tuple[str, str, Direction]] = {}
        self.start_state: str = ""
        self.accept_states: set = set()
        self.reject_states: set = set()
        self.blank_symbol: str = "_"
    
    def set_tape(self, input_string: str):
        """初始化紙帶"""
        self.tape = list(input_string) + [self.blank_symbol]
        self.head = 0
    
    def step(self) -> bool:
        """
        執行一步
        返回 True 如果機器繼續運行，False 如果停機
        """
        current_symbol = self.tape[self.head]
        key = (self.state, current_symbol)
        
        if key not in self.transitions:
            return False  # 無定義的轉換，停機
        
        new_state, new_symbol, direction = self.transitions[key]
        
        self.tape[self.head] = new_symbol
        self.state = new_state
        
        if direction == Direction.LEFT:
            if self.head > 0:
                self.head -= 1
            else:
                self.tape.insert(0, self.blank_symbol)
        elif direction == Direction.RIGHT:
            self.head += 1
            if self.head >= len(self.tape):
                self.tape.append(self.blank_symbol)
        
        return True
    
    def run(self, max_steps: int = 10000) -> bool:
        """
        運行圖靈機直到停機
        返回 True 如果接受，False 如果拒絕
        """
        steps = 0
        while steps < max_steps:
            if self.state in self.accept_states:
                return True
            if self.state in self.reject_states:
                return False
            if not self.step():
                # 無定義轉換，檢查是否接受/拒絕
                return self.state in self.accept_states
            steps += 1
        return False  # 超時
    
    def get_tape_string(self) -> str:
        """獲取紙帶內容"""
        return ''.join(c for c in self.tape if c != self.blank_symbol)


class PalindromeTM(TuringMachine):
    """識別二進制回文的圖靈機"""
    
    def __init__(self):
        super().__init__()
        
        self.states = {'q0', 'q1', 'q2', 'q3', 'q4', 'accept', 'reject'}
        self.alphabet = {'0', '1'}
        self.tape_alphabet = {'0', '1', '_'}
        self.start_state = 'q0'
        self.accept_states = {'accept'}
        self.reject_states = {'reject'}
        
        # 轉換函數
        # 格式：(狀態, 讀取) -> (新狀態, 寫入, 方向)
        self.transitions = {
            # q0: 從左端讀取，標記並向右
            ('q0', '0'): ('q1', 'X', Direction.RIGHT),
            ('q0', '1'): ('q2', 'X', Direction.RIGHT),
            ('q0', 'X'): ('q0', 'X', Direction.RIGHT),
            
            # q1: 向右跳過標記和空格，找右端
            ('q1', '0'): ('q1', '0', Direction.RIGHT),
            ('q1', '1'): ('q1', '1', Direction.RIGHT),
            ('q1', 'X'): ('q1', 'X', Direction.RIGHT),
            ('q1', '_'): ('q3', '_', Direction.LEFT),
            
            # q2: 同上，用於 '1'
            ('q2', '0'): ('q2', '0', Direction.RIGHT),
            ('q2', '1'): ('q2', '1', Direction.RIGHT),
            ('q2', 'X'): ('q2', 'X', Direction.RIGHT),
            ('q2', '_'): ('q4', '_', Direction.LEFT),
            
            # q3: 比較 '0'
            ('q3', '0'): ('reject', '0', Direction.STAY),
            ('q3', '1'): ('q3', '1', Direction.LEFT),
            ('q3', 'X'): ('q0', 'X', Direction.RIGHT),
            
            # q4: 比較 '1'
            ('q4', '1'): ('reject', '1', Direction.STAY),
            ('q4', '0'): ('q4', '0', Direction.LEFT),
            ('q4', 'X'): ('q0', 'X', Direction.RIGHT),
            
            # 接受所有 X 和空
            ('q0', '_'): ('accept', '_', Direction.STAY),
            ('q3', '_'): ('accept', '_', Direction.STAY),
            ('q4', '_'): ('accept', '_', Direction.STAY),
        }


def test_palindrome():
    """測試回文識別器"""
    print("=== 回文識別圖靈機測試 ===\n")
    
    test_cases = [
        ("101", True),   # 是回文
        ("11", True),    # 是回文
        ("0", True),     # 是回文
        ("", True),     # 空串是回文
        ("1010", False), # 不是回文
        ("10", False),   # 不是回文
        ("1001", True),  # 是回文
    ]
    
    for input_str, expected in test_cases:
        tm = PalindromeTM()
        tm.set_tape(input_str)
        result = tm.run()
        
        status = "✓" if result == expected else "✗"
        print(f"  {status} '{input_str}': {'接受' if result else '拒絕'} (期望: {'接受' if expected else '拒絕'})")


test_palindrome()
```

  ✓ '': 接受 (期望: 接受)
  ✓ '0': 接受 (期望: 接受)
  ✓ '1': 接受 (期望: 接受)
  ✓ '11': 接受 (期望: 接受)
  ✓ '00': 接受 (期望: 接受)
  ✓ '101': 接受 (期望: 接受)
  ✓ '111': 接受 (期望: 接受)
  ✓ '1010': 拒絕 (期望: 拒絕)
  ✓ '10': 拒絕 (期望: 拒絕)
  ✓ '1001': 接受 (期望: 接受)

## 6.5 通用圖靈機

### 6.5.1 圖靈的天才洞見

圖靈論文中最重要的洞見不是圖靈機本身，而是**通用圖靈機**（Universal Turing Machine, UTM）的概念。

一台通用圖靈機可以**模擬任何其他圖靈機**！

這意味著：
- 可以用一台「通用」機器執行任何演算法
- 硬體和軟體的概念是分離的
- 這是現代電腦的理論基礎

### 6.5.2 通用圖靈機的原理

通用圖靈機的原理很簡單：
1. 將目標圖靈機的描述（狀態、轉換函數等）編碼到紙帶上
2. 將目標圖靈機的輸入放在紙帶上
3. 按照目標圖靈機的行為方式運作

```
┌────────────────────────────────────────────────────────────────┐
│                     通用圖靈機原理                              │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  輸入格式：                                                     │
│  ┌─────────────┬─────────────────────┐                        │
│  │ 機器描述 M  │     輸入字串 w       │                        │
│  └─────────────┴─────────────────────┘                        │
│                                                                │
│  M 用哥德爾編號表示                                             │
│  U 可以模擬 M(w) 的執行                                         │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 6.5.3 與現代電腦的類比

通用圖靈機的概念預示了現代電腦的結構：

| 圖靈機概念 | 現代電腦 |
|-----------|---------|
| 紙帶 | 記憶體 |
| 讀寫頭 | CPU/記憶體控制器 |
| 狀態寄存器 | 程式計數器 |
| 轉換函數 | 指令集架構 |
| 通用圖靈機 | 儲存程式電腦 |

馮·諾依曼結構實際上就是通用圖靈機的物理實現！

## 6.6 停機問題

### 6.6.1 問題定義

**停機問題**（Halting Problem）問的是：

> 給定一個程式 P 和輸入 I，是否存在一個演算法，可以判斷 P(I) 是否會停機？

### 6.6.2 圖靈的證明

圖靈用反證法證明了停機問題是不可判定的：

假設存在一個演算法 H，可以判定任意程式是否停機。

構造一個新的程式：

```
PROGRAM DREAM(P):
    if H(P, P) = "停機" then
        無限循環  # 永遠不停機
    else
        停機
```

現在問：DREAM(DREAM) 會停機嗎？

- 如果停機 → H 說「停機」，但 DREAM 無限循環 → 矛盾
- 如果不停機 → H 說「不停機」，但 DREAM 停機 → 矛盾

因此 H 不存在，停機問題不可判定。

### 6.6.3 停機問題的影響

停機問題的直接後果是：

```
┌────────────────────────────────────────────────────────────────┐
│                  停機問題的影響                                 │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  定理：不存在能夠判定任意程式行為的通用程式                      │

```

## 6.7 圖靈機與λ演算的等價性

### 6.7.1 兩種計算模型

1936年，丘奇和圖靈幾乎同時提出了兩種計算模型：
- 丘奇：λ演算
- 圖靈：圖靈機

這兩種模型是**計算等價的**——它們可以計算完全相同的函數。

### 6.7.2 證明思路

**λ演算 → 圖靈機**：
- 每個λ表達式都可以翻譯為圖靈機
- 圖靈機可以模擬β歸約

**圖靈機 → λ演算**：
- 每台圖靈機都可以用λ表達式編碼
- λ演算可以模擬圖靈機的狀態轉換

這個等價性是**丘奇-圖靈論題**的基礎。
