# 第五十八章：Deutsch算法

## 58.1 量子算法的起源

量子計算是物理學和計算機科學交叉的前沿領域。1982年，理查德·費曼在一次演講中首次提出了一個深刻的問題：經典計算機模擬量子系統需要指數級的時間和資源，那麼量子系統本身能否成為計算機？

這個問題催生了量子計算領域。費曼的洞見揭示了一個重要的對稱性：如果自然是量子力學的，那麼模擬自然的最佳方式可能是使用量子力學系統本身。

1985年，大衛·杜茲（David Deutsch）在牛津大學發表了開創性論文，提出了量子圖靈機的概念，並設計了第一個量子算法——Deutsch算法。這是量子計算領域的里程碑，展示了量子計算機在解決某些問題時可以超越經典計算機。

Deutsch算法的意義在於它是第一個展示量子計算潛在優勢的算法。雖然這個問題本身可能看起來很簡單（甚至有些人為），但它為後續更重要的量子算法（如Shor算法和Grover算法）奠定了基礎。

## 58.2 Deutsch問題的陳述

Deutsch問題是量子計算中展示量子優勢的最簡單例子。問題陳述如下：

設$f: \{0,1\} \to \{0,1\}$是一個黑盒子函數。這個函數有四種可能的類型：

1. **常數0**：$f(0) = 0, f(1) = 0$
2. **常數1**：$f(0) = 1, f(1) = 1$
3. **平衡0-1**：$f(0) = 0, f(1) = 1$
4. **平衡1-0**：$f(0) = 1, f(1) = 0$

問題：判斷$f$是「常數函數」（類型1或2）還是「平衡函數」（類型3或4）。

注意：我們不需要區分是哪個具體函數，只需要判斷函數的「類型」。

## 58.3 經典複雜度分析

讓我們分析這個問題的經典複雜度。

經典上，我們需要查詢函數$f$來獲得信息。每次查詢可以得到$f(0)$或$f(1)$的值。

**單次查詢**：如果我們只查詢一次，例如$f(0)$，我們能確定什麼？
- 如果$f(0) = 0$，函數可能是常數0或平衡0-1——不確定
- 如果$f(0) = 1$，函數可能是常數1或平衡1-0——不確定

因此，單次查詢不足以解決問題。我們至少需要兩次查詢，檢查$f(0)$和$f(1)$的值。

**結論**：經典算法需要恰好兩次查詢才能保證正確解決Deutsch問題。

## 58.4 量子算法電路

Deutsch量子算法只需要一次查詢就能解決問題！這是量子優勢的第一個明確例子。

量子算法的步驟如下：

**步驟1：準備初始狀態**
$$|0\rangle|1\rangle \to (H \otimes H) \to \frac{1}{2}(|0\rangle + |1\rangle)(|0\rangle - |1\rangle)$$

**步驟2：應用Oracle**
$$U_f |x\rangle|y\rangle = |x\rangle|y \oplus f(x)\rangle$$

當第二個量子位元是$|1\rangle$時：
$$U_f |x\rangle|1\rangle = |x\rangle|1 \oplus f(x)\rangle = (-1)^{f(x)} |x\rangle|1\rangle$$

**步驟3：對第一個量子位元應用Hadamard**
$$(H \otimes I) \left[ \frac{1}{\sqrt{2}} \left( (-1)^{f(0)}|0\rangle + (-1)^{f(1)}|1\rangle \right) |1\rangle \right]$$

**步驟4：測量**
- 如果$f(0) = f(1)$（常數函數），狀態為$\pm|0\rangle|1\rangle$，測量得到$|0\rangle$
- 如果$f(0) \neq f(1)$（平衡函數），狀態為$\pm|1\rangle|1\rangle$，測量得到$|1\rangle$

## 58.5 算法正確性的證明

讓我們詳細證明Deutsch算法的正確性。

定義Oracle $U_f$：$U_f|x\rangle|y\rangle = |x\rangle|y \oplus f(x)\rangle$

初始狀態：$|\psi_0\rangle = |0\rangle|1\rangle$

第一步：對兩個量子位元應用Hadamard：
$$|\psi_1\rangle = \frac{1}{\sqrt{2}}(|0\rangle + |1\rangle) \cdot \frac{1}{\sqrt{2}}(|0\rangle - |1\rangle) = \frac{1}{2}(|00\rangle - |01\rangle + |10\rangle - |11\rangle)$$

第二步：應用Oracle $U_f$（注意：$|0\rangle|1\rangle \to |0\rangle|1\oplus f(0)\rangle = (-1)^{f(0)}|0\rangle|1\rangle$，因為$|1\oplus 0\rangle = |1\rangle$，$|1\oplus 1\rangle = |0\rangle = -|1\rangle$乘以$-1$）

$$|\psi_2\rangle = \frac{1}{2} \left[ (-1)^{f(0)}(|00\rangle - |01\rangle) + (-1)^{f(1)}(|10\rangle - |11\rangle) \right]$$

第三步：對第一個量子位元應用Hadamard：
$$|\psi_3\rangle = \frac{1}{2\sqrt{2}} \left[ (-1)^{f(0)}|0\rangle(|0\rangle - |1\rangle) + (-1)^{f(1)}|1\rangle(|0\rangle - |1\rangle) \right]$$

整理後：
$$|\psi_3\rangle = \frac{1}{2\sqrt{2}} \left[ \left( (-1)^{f(0)} + (-1)^{f(1)} \right) |0\rangle - \left( (-1)^{f(0)} - (-1)^{f(1)} \right) |1\rangle \right] |1\rangle$$

分析：
- 如果$f(0) = f(1)$，第一項係數為$\pm 2$，第二項為$0$，狀態為$\pm|0\rangle|1\rangle$
- 如果$f(0) \neq f(1)$，第一項為$0$，第二項為$\pm 2$，狀態為$\pm|1\rangle|1\rangle$

測量第一個量子位元：如果得到$|0\rangle$，則$f$是常數；如果得到$|1\rangle$，則$f$是平衡。

## 58.6 量子平行性的展現

Deutsch算法展示了量子平行性的威力。讓我們理解這個算法的量子優勢來源。

在量子算法中，我們對初始狀態$\frac{1}{\sqrt{2}}(|0\rangle + |1\rangle)$應用Oracle。這實際上同時計算了$f(0)$和$f(1)$：

$$U_f \frac{1}{\sqrt{2}}(|0\rangle + |1\rangle)|1\rangle = \frac{1}{\sqrt{2}}((-1)^{f(0)}|0\rangle + (-1)^{f(1)}|1\rangle)|1\rangle$$

這個狀態同時包含了$f(0)$和$f(1)$的信息！這就是量子平行性——一次操作同時處理了兩個輸入。

然而，量子平行性本身並不能直接給出優勢。我們不能直接「讀取」兩個函數值——測量會導致波函數坍縮，只得到一個結果。

真正的技巧是干涉。Hadamard閘將兩個不同相位的信息「混合」在一起，使得不同類型函數的最終狀態在正交方向上。通過測量，我們可以高置信度地區分這兩種類型。

## 58.7Deutsch-Jozsa推廣

Deutsch算法可以被推廣到多量子位元的情況，稱為Deutsch-Jozsa算法。

問題推廣：對於$n$位輸入的函數$f: \{0,1\}^n \to \{0,1\}$，已知它要么是：
- 常數函數：對所有輸入，$f(x) = 0$或$f(x) = 1$
- 平衡函數：對一半輸入$f(x) = 0$，對另一半$f(x) = 1$

任務：確定$f$是常數還是平衡。

**經典複雜度**：在最壞情況下，經典算法需要$2^{n-1} + 1$次查詢才能確定答案。

**量子複雜度**：Deutsch-Jozsa算法只需要1次查詢！

電路結構：
1. 準備$n+1$個量子位元：$|0\rangle^{\otimes n}|1\rangle$
2. 對所有量子位元應用Hadamard
3. 應用Oracle $U_f$
4. 對$n$個輸入量子位元應用Hadamard
5. 測量$n$個輸入量子位元

如果$f$是常數，測量結果恆為$|0\rangle^{\otimes n}$。如果$f$是平衡，測量結果永遠不是$0\rangle^{\otimes n}$。

## 58.8 Bernstein-Vazirani算法

Bernstein-Vazirani算法是Deutsch-Jozsa算法的進一步推廣，展示了量子計算對某些確定性問題的指數加速。

問題：設存在一個隱藏位串$a \in \{0,1\}^n$，函數$f(x) = a \cdot x \mod 2$（點積）。給定$f$，找出$a$。

**經典複雜度**：經典上需要$n$次查詢——每次查詢一個位元。

**量子複雜度**：Bernstein-Vazirani算法只需要1次查詢！

電路：
1. 準備$|0\rangle^{\otimes n}|1\rangle$
2. 應用Hadamard到所有量子位元
3. 應用Oracle
4. 再次應用Hadamard
5. 測量，結果直接給出$a$

這個結果令人驚訝：量子算法1次查詢解決的問題，經典需要$n$次查詢。對於大的$n$，這是顯著的加速。

## 58.9 Simon算法

Simon算法是量子計算對經典計算指數加速的另一個重要例子，比Shor算法早几年。

問題：設存在一個隱藏弦$s \in \{0,1\}^n$，函數$f: \{0,1\}^n \to \{0,1\}^n$滿足：對於所有$x, y$，$f(x) = f(y)$當且僅當$x \oplus y \in \{0^n, s\}$。換句話說，$f$在兩個陪集上恆定。

任務：找出$s$。

**經典複雜度**：經典上需要指數級查詢。

**量子複雜度**：Simon算法只需要$O(n)$次查詢。

量子算法利用量子傅立葉變換來找到與$s$正交的向量。通過重複測量得到足夠多的正交向量，可以確定$s$。

Simon算法的量子加速後來啟發了Shor算法的設計。

## 58.10 量子算法的意義

Deutsch算法及其推廣具有深遠的理論意義：

**第一個量子優勢證明**：Deutsch-Jozsa算法是首個明確展示量子計算可以超越經典計算的例子。

**複雜性理論洞察**：這些算法揭示了量子計算在處理「黑盒子函數」問題時的強大能力。

**電路模型驗證**：Deutsch算法可以用很少的量子位元實現，為量子硬體提供了重要的測試案例。

**算法設計啟示**：Deutsch-Jozsa、Simon和Shor算法的設計思路一脈相承——利用量子傅立葉變換來提取週期性信息。

## 58.11 實驗實現

Deutsch算法已在多種量子系統中得到實驗驗證：

**光子系統**：使用光子的偏振和路徑自由度編碼量子位元，是最早實現Deutsch算法的系統之一。

**NMR量子計算**：核磁共振系統使用分子中的核自旋作為量子位元，實現了早期的Deutsch算法演示。

**超導量子電路**：IBM和Google的超導量子處理器已實現Deutsch算法的演示。

**離子阱**：離子阱系統以其高門精度和長相干時間，已實現Deutsch算法的精確演示。

這些實驗驗證了量子算法的理論預言，儘管在噪聲和規模上仍有挑戰。

Deutsch量子算法只需要一次查詢就能解決問題，而經典算法需要兩次。這種「確定性」的量子優勢雖然看似微小，但具有重要的理論意義。

然後，我們將Deutsch算法推廣到Deutsch-Jozsa算法（多量子位元版本）和Bernstein-Vazirani算法，展示了更強大的量子加速。

Simon算法作為連接這些早期量子算法與Shor算法的橋樑，展示了更顯著的指數加速。
